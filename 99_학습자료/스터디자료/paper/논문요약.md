# 1. 반도체 WT → Field 불량 예측 (4편)

## 1-1. Prediction of Field Failure Rate Using Data Mining in Automotive Semiconductor (2018)

### 논문 정보
- **저자**: Gyungsik Yun (Texas Instruments Korea), Hee-Won Jung, Sungbum Park (Hoseo University)
- **학회**: Procedia Computer Science 139, ITQM 2018

### 연구 배경 및 목적
자동차용 반도체는 수명 15년 이상, 온도 -40~150도C, 불량률 0% 목표 등 극히 높은 신뢰성 요구사항을 가진다. 기존 JEDEC/AEC 표준 기반 가속 수명 시험은 고객의 다양한 사용 조건을 반영하지 못한다. 반도체 조립(assembly) 공정과 패키지 테스트(package test) 데이터를 데이터마이닝으로 분석하여 **잠재적 Field 불량을 예측**하는 모델을 제안한다.

### 데이터
- 외국계 반도체 회사(A사)의 3년간 실생산 데이터
- **86개 품질 지표** (조립 공정 + 패키지 테스트)
- 총 **1,307개 Lot**: 불량 발생 132개, 미발생 1,175개 (불균형 비율 8.9:1)
- 2-class 분류 문제 (Field 불량 발생/미발생)

### 방법론
- 트리 기반 알고리즘 4종 비교: Decision Tree, **Random Forest**(50개 트리), ExtraTrees, AdaBoost
- 3-Fold Cross-Validation
- RF Feature Importance로 상위 9개 핵심 변수 선별 후 2개씩 쌍으로 추출하여 Decision Boundary 시각화

### 핵심 결과

| 모델 | 학습 정확도 | 테스트 정확도 |
|------|-----------|-------------|
| Random Forest | 100% | **92.8%** |
| Zero-R (다수 클래스 예측) | - | 89.7% |

**Feature Importance Top 9**: 패키지 테스트 공정 변수 6개 + 조립 공정 변수 3개. 조립 공정 불량률이 높은 Lot은 열적/물리적 스트레스가 상대적으로 높아 Field 불량 확률 증가. **ExtraTrees가 모든 feature 쌍에서 최고 성능**.

### 우리 프로젝트 적용 시사점
- **문제 구조 동일**: 패키지 테스트/조립 데이터 → Field 불량 예측 ≈ WT 데이터 → RCC 예측
- **ExtraTrees의 우수성**: baseline 모델 비교에 ExtraTrees 포함 타당성 확인
- **Feature Importance 기반 선별** 필수: 86개에서 9개 핵심 변수만으로 충분
- 결론에서 "회귀 확장 필요" 언급 → Two-Stage(분류+회귀) 접근법 타당성 확인

## 1-2. Applying ML Models on Metrology Data for Predicting Device Electrical Performance (2023)

### 논문 정보
- **저자**: Bappaditya Dey 외 (imec, Heriot-Watt University)
- **학회**: arXiv:2312.09462

### 연구 배경 및 목적
3nm 이하 반도체 공정에서 Overlay 오차가 최종 전기적 성능(정전용량)에 미치는 영향을 정량화하고, 초기 공정 단계의 계측 데이터로 **최종 전기적 성능을 ML로 조기 예측**하여 불량 웨이퍼의 rework/scrap 조기 결정을 지원한다.

### 데이터
- imec N-14 BEOL 공정, 4장 웨이퍼 (2장 Non-Programmed + 2장 Programmed Overlay)
- CD 예측: **30개 입력 피처** (26개 overlay + die 위치 X,Y + 타겟 위치 X,Y), 웨이퍼당 700+ 데이터포인트
- 정전용량 예측: **29개 입력 피처**, 80:20 train/test 분할

### 방법론
- 4종 ML 회귀 모델 비교: Linear Regression, SVR(RBF), Random Forest, **Extra Tree Regressor**
- 정규화(단위 분산 스케일링) 적용
- **DBSCAN** 알고리즘으로 타겟 변수 이상치 탐지 및 클리닝

### 핵심 결과

**CD 예측 (AEI DBO, AB1):**

| 모델 | R² (test) | MSE |
|------|----------|-----|
| Linear Regression | 0.836 | 1.168 |
| SVR | 0.853 | 1.045 |
| Random Forest | 0.876 | 0.881 |
| **Extra Tree Regressor** | **0.891** | **0.779** |

**정전용량 예측**: DBSCAN 적용 후 R² 0.742 → **0.950**, MSE 182.100 → **0.077** (극적 개선)

### 우리 프로젝트 적용 시사점
- **Extra Tree Regressor가 모든 실험에서 RF보다 우수** → baseline에 ExtraTrees 반드시 포함
- **DBSCAN 기반 이상치 클리닝**: health 데이터의 max=1.0 등 극단값에 DBSCAN 탐지 고려
- **Die 위치 피처(X, Y)가 성능 향상에 기여** → run_wf_xy에서 die_x, die_y 파싱 (Stage 4.5)

## 1-3. Rare Class Prediction Model for Smart Industry in Semiconductor Manufacturing (2024)

### 논문 정보
- **저자**: Abdelrahman Farrag, Mohammed-Khalil Ghali, Yu Jin (SUNY Binghamton)
- **학회**: arXiv:2406.04533

### 연구 배경 및 목적
반도체 제조 센서 데이터의 **심각한 클래스 불균형**(1:14), **높은 결측률**(4.5%), **노이즈 피처**(591개) 세 가지 난제를 동시에 해결하여 희소 클래스(불량) 예측 정확도를 높이는 구조화된 접근법 개발.

### 데이터
- **SECOM 데이터셋** (UCI ML Repository): 1,567개 샘플 × 591개 센서 측정값
- 불량 샘플 104개 (클래스 불균형 1:14), 결측률 4.5%

### 방법론

**전처리 순서:**
1. 결측률 50% 초과 피처 제거 → 28개 삭제
2. 상수값 피처 제거 → 116개 제거
3. k-NN imputation (최고 클래스 분리 달성)
4. 상관계수 70% 초과 피처 제거 → **204개**로 축소

**투표 기반 피처 선택 (핵심 기여):**
12가지 피처 선택 알고리즘(ANOVA, Mutual Information, **Boruta**, MARS, RFE by LR/SVM/RF, LASSO, Sequential FS by XGB/SVM 등)의 투표로 **최소 3개 이상에서 선택된 81개 피처** 최종 채택

**리샘플링**: SMOTE oversampling(소수 70%) + undersampling(다수 80%) 결합이 최적

**모델**: LR, SVM, DTC, RF, GBC, **XGBoost** 6종 비교

### 핵심 결과

| 시나리오 | 모델 | AUC | Recall | Precision | FAR |
|---------|------|-----|--------|-----------|-----|
| Combined (최적) | XGBoost | **0.95** | **0.96** | 0.66 | **0.03** |

기존 연구(Deep NN+PSO: Recall 0.98, In-painting KNN: 0.78) 대비 Recall과 AUC에서 개선, FAR 최저.

### 우리 프로젝트 적용 시사점
- **투표 기반 피처 선택**: Boruta + LightGBM importance + Null Importance + Permutation Importance 결과를 교차 검증(투표식)하여 최종 피처 선정
- **전처리 순서**: 결측률 50% 초과 제거 → 상수값 제거 → k-NN imputation → Stage 1~2 전략과 일치
- **SMOTE + undersampling 결합**: Two-Stage 모델의 Stage 1(Y=0 vs Y>0 분류) 학습 시 적용
- **XGBoost의 일관된 우수성**: 불균형 데이터에서 XGBoost가 최고 → Stage 1 분류기에 사용 근거

## 1-4. Yield Prediction in Semiconductor Manufacturing Using Two-Step Machine Learning (2025)

### 논문 정보
- **저자**: Rebecca Busch 외 (University of Siegen, Elmos Semiconductor SE)
- **학회**: International Journal of Production Research, Taylor & Francis

### 연구 배경 및 목적
**FDC(Fault Detection and Classification) 데이터**만으로 생산 초기 단계에서 최종 전기적 수율을 예측하는 최초의 시도. **이상치(불량 웨이퍼)를 먼저 분류하여 제거한 뒤 양품만으로 수율 예측**하는 Two-Step 접근법 제안.

### 데이터
- 주요 파운드리(Elmos Semiconductor SE)의 실생산 데이터, 350nm 노드
- **21,630개 웨이퍼**, 6개월간 수집, **1,188개 → 전처리 후 804개 파라미터**
- 학습 15,140개 / 테스트 6,490개

### 방법론

**Boruta(WFS) 피처 선택**: 804개 → **76개** 파라미터로 축소 (비교: 상관 필터링 352개, PCA는 성능 열위)

**Two-Step 모델:**
- **Step 1 (분류)**: Random Forest로 3개 클래스 분류 (acceptable/average/poor). 정확도 **0.990**.
- **Step 2 (회귀)**: acceptable로 분류된 데이터만으로 수율 예측

**5종 ML 비교**: RF, Neural Networks, KNN, GLM, GLMBoost

### 핵심 결과

| 모델 | 테스트 데이터 | R² | RMSE | MAE |
|------|------------|-----|------|-----|
| m1 (전체, 804 params) | 전체 | 0.192 | 0.031 | 0.012 |
| m2 (acceptable, 804) | 실제 acceptable | **0.496** | 0.130 | 0.009 |
| m4 (acceptable + WFS, 76) | **실제 acceptable** | **0.538** | **0.011** | **0.008** |

- **Two-Step**: R² 0.192 → **0.496** (전체 대비 실제 acceptable)
- **Boruta 추가**: R² 0.496 → **0.538** (804→76개에서 오히려 향상)
- **RF가 모든 조건에서 최고 성능**

### 우리 프로젝트 적용 시사점
- **Two-Stage 전략의 직접적 근거**: 분류→회귀 구조가 R²를 0.192→0.538로 극적 향상. 우리의 Y=0 vs Y>0 분류 → Y>0 회귀 전략과 동일
- **Boruta의 피처 선택 효과**: 804→76개로 축소하면서 R²가 오히려 향상. 노이즈 피처 제거가 핵심
- **분류 정확도의 중요성**: 1% 미만의 분류 오류도 회귀 성능을 크게 저하 → Stage 1 분류기 최적화 필수
- **PCA는 부적합**: 해석 불가능하고 성능 열위. Boruta 등 해석 가능한 피처 선택 방법이 적합

# 2. Zero-Inflated 데이터 처리 (3편)

## 2-1. Hybrid ML Approach to Zero-Inflated Data Improves Accuracy of Dengue Prediction (2024)

### 논문 정보
- **저자**: Micarnaldo Ernesto Francisco, Thaddeus M. Carvajal, Kozo Watanabe 외
- **학회**: PLOS Neglected Tropical Diseases

### 연구 배경 및 목적
뎅기열 발생률 예측에서 fine spatiotemporal resolution(주간-마을 단위)의 데이터는 **72% 이상이 0**이어서 ML 모델이 0을 과도하게 예측한다. 기존 hurdle/ZI 모델이 이산(count) 데이터에 한정된 반면, **연속 데이터에 적용 가능한 Hybrid(분류+회귀) 접근법**을 제안한다.

### 데이터
- 필리핀 Metro Manila, 1,806개 마을, 2009~2013 (train), 2014 (test)
- **종속변수**: 주간 뎅기 발생률 (연속 변수, log(n+1) 변환)
- **30개 독립변수**: 기후, 토지이용, 홍수 위험도 등
- **Zero 비율**: training 71.4%, test 65.8%

### 방법론

**6종 ML 비교**: GAM, RF, CIF(Conditional Inference Forest), ANN, SVR, XGB

**Hybrid 2단계 접근법 (핵심):**
1. **Stage 1 (Qualitative/분류)**: **RF**로 뎅기 발생 여부 예측 (0 vs 1). AUC=**0.899**
2. **Stage 2 (Quantitative/회귀)**: "발생(1)"으로 예측된 위치-시점만 **CIF**로 정량 예측
3. Stage 1에서 0 예측 → 최종 0, Stage 1에서 1 예측 → Stage 2 회귀값 출력

**HAI (Hybrid Accuracy Index)**: AUC × 0.7 + S-R²ₐ × 0.3 (zero 비율에 비례한 가중치)

### 핵심 결과

| 접근법 | 정량 예측 R² | 비고 |
|--------|------------|------|
| Quantitative 단독 (CIF) | 0.117 (S-R²ₐ) | zero-inflation으로 인해 극히 낮음 |
| Hybrid (RF분류 + CIF회귀) | **~0.716** | Stage 1에서 0 필터링 후 R² 대폭 개선 |

- **Aggregation으로 zero 비율 감소**: 공간 해상도를 낮추면(village→city) zero가 줄어 정량 모델 정확도 향상

### 우리 프로젝트 적용 시사점
- **Two-Stage 구조 직접 적용 가능**: 우리 health data도 70.8%가 0이고 **연속값**이므로 이 논문의 hybrid 접근법이 정확히 동일 구조
  - Stage 1: Classifier로 Y=0 vs Y>0
  - Stage 2: Y>0 예측된 샘플만 Regressor로 health 값 예측
  - 최종: P(Y>0) × E[Y|Y>0]
- **각 Stage에 다른 알고리즘 배치**: 1단계(RF) ≠ 2단계(CIF). 우리도 Stage 1에 LightGBM, Stage 2에 XGBoost 등 조합 실험 필요
- **연속 데이터에 Hybrid 적용 가능 확인**: 기존 hurdle/ZI 모델은 count 전용이지만, hybrid 방식은 연속값에도 적용 가능

## 2-2. An Interpretable ML-Based Hurdle Model for Zero-Inflated Data Analysis (2024)

### 논문 정보
- **저자**: Moataz Bellah Ben Khedher, Dukgeun Yun
- **학회**: Applied Sciences (MDPI), Vol. 14, No. 23

### 연구 배경 및 목적
교통사고 빈도 데이터(81% zeros, overdispersion)에서 **CatBoost 기반 ML Hurdle 모델**을 개발하고, **Custom Loss Function**으로 class imbalance 대응, **SHAP 분석**으로 해석 가능성 확보.

### 데이터
- 한국도로공사(KICT), 2011~2015년 일반국도
- **8,636개 도로 구간**, 총 3,182건 사고
- **Zero 비율**: 81.7% (7,052 구간이 사고 0건)
- 독립변수: 구간 길이, 경사도, 곡률, 차로폭, AADT 등

### 방법론

**수학적 정의:**
```
최종 예측: E[Y|X] = phi_hat(x) × mu_hat(x)
  phi_hat(x) = P(Y>0)  (분류 모델 출력)
  mu_hat(x) = E[Y|Y>0]  (회귀 모델 출력)
```

**Stage 1 (CatBoost Classifier):**
- **Custom Weighted Log-Loss**: `w_Cj = 1/n_Cj` (클래스 빈도 역수)로 소수 클래스에 높은 가중치

**Stage 2 (CatBoost Regressor):**
- **Custom Poisson Loss**: `L = -Σ(yi·log(ŷi) - ŷi)` → count/비음수 데이터 특성에 적합
- Y>0 데이터만으로 학습 (zero-truncated)

### 핵심 결과

**Classification Stage에서 Custom Loss 효과:**

| 모델 | Recall | F1 |
|------|--------|-----|
| CatBoost (Default) | 0.26 | 0.35 |
| **CatBoost (Custom Loss)** | **0.77** | **0.78** |

**Full Model 비교:**

| 모델 | RMSE | MAE |
|------|------|-----|
| Poisson Hurdle | 2.097 | 0.527 |
| NB Hurdle | 1.336 | 0.511 |
| CatBoost (Single Stage) | 1.197 | 0.421 |
| **CatBoost Hurdle (Proposed)** | **0.978** | **0.346** |

- Single-stage 대비 RMSE **18.3% 감소**, NB Hurdle 대비 **26.8% 감소**

### 우리 프로젝트 적용 시사점
- **Custom Loss Function의 중요성**: Classification에서 weighted log-loss로 recall 0.26→0.77 달성. 우리도 Stage 1에서 class_weight 또는 custom loss 적용 필수
- **Regression에서 Poisson Loss > MSE**: 우리도 Stage 2에서 Poisson/Tweedie loss 실험 필요
- **SHAP 기반 Stage별 해석**: 어떤 WT feature가 "불량 발생 여부"(Stage 1)와 "불량 규모"(Stage 2)에 기여하는지 분리 해석 가능 → 공정 개선 인사이트 도출
- **Single-stage vs Two-stage**: 18.3% RMSE 개선 → Stage 7-B(Two-Stage Model)의 기대 효과 정량적 뒷받침

## 2-3. Yield Prediction via Spatial Modeling of Clustered Defect Counts Across a Wafer Map (2007)

### 논문 정보
- **저자**: Suk Joo Bae (한양대), Jung Yoon Hwang (삼성전자), Way Kuo (U of Tennessee)
- **학회**: IIE Transactions, Vol. 39, No. 12

### 연구 배경 및 목적
웨이퍼 결함은 공간적으로 클러스터링되며 zero-defect chip이 과다하다. 기존 Poisson(균일 분포 가정→수율 과소추정)과 Negative Binomial(공간 패턴 무시)의 한계를 극복하기 위해, IC 칩의 **공간 좌표(radial, angular)를 공변량**으로 사용하는 **Zero-Inflated Poisson(ZIP) regression** 제안.

### 데이터
- 20×20 = 400개 IC 칩 (하나의 웨이퍼)
- **종속변수**: 각 칩의 결함 수 (0~12개, count data)
- **Zero 비율**: 318/400 = **79.5%** → zero-inflated
- **독립변수**: 극좌표 {r, cos(φ), sin(φ), r·cos(φ), r·sin(φ)}

### 방법론

**ZIP 모델의 두 가지 zero 소스:**
```
P(N=0) = π + (1-π)·exp(-Λ)     [구조적 zero + 우연적 zero]
P(N=k) = (1-π)·exp(-Λ)·Λᵏ/k!   (k≥1)
```
- **π (구조적 zero)**: 결함 발생이 불가능한 완벽 상태의 칩 확률
- **(1-π)·exp(-Λ) (우연적 zero)**: 결함 가능하나 우연히 0

**두 개의 regression:**
```
log(Λ) = 1.4627 - 0.0716·r + 0.0986·cos(φ)     [Poisson part: 결함 수]
logit(p) = 1.3094 - 0.2764·sin(φ) + 0.1044·r·sin(φ)  [Zero part: 완벽 확률]
```

### 핵심 결과

| 모델 | 예측 수율 | 관측 수율 | 오차 |
|------|----------|----------|------|
| Poisson (공간X) | 52.33% | 79.5% | -27.17% |
| NB (공간O) | 79.32% | 79.5% | -0.18% |
| **ZIP (공간O)** | **79.53%** | 79.5% | **+0.03%** |

- **ZIP이 가장 정확** (오차 0.03%), overdispersion도 1.620으로 NB(7.546)보다 낮음

### 우리 프로젝트 적용 시사점
- **공간 피처의 결정적 중요성**: 공간 정보 없는 Poisson(오차 27%) vs ZIP+공간(오차 0.03%). Stage 4.5의 radial_dist, die_x, die_y가 유사 역할
- **두 가지 zero 소스 개념**: Y=0 중 "진짜 양품"과 "잠재 불량"의 구분 → Two-Stage의 Stage 1이 이 구분을 학습
- **피처 엔지니어링 + 모델 구조의 시너지**: 공간 피처 + zero 분리 결합이 압도적 성능 차이를 만듦

# 3. 공간 피처 / Wafer Map (3편)

## 3-1. Wafer Map Failure Pattern Classification Using Geometric Transformation-Invariant CNN (2023)

### 논문 정보
- **저자**: Iljoo Jeong 외 (POSTECH, 연세대)
- **학회**: Scientific Reports (Nature) 13:8127

### 연구 배경 및 목적
웨이퍼 맵 결함 패턴 분류에서 기존 CNN은 **회전/뒤집기 불변성**을 갖추지 못해 데이터 증강에 의존한다. **Radon 변환 + 커널 뒤집기(Kernel Flip)**를 결합하여 모델 아키텍처 자체에 기하학적 불변성을 내장, 소량 데이터에서도 높은 성능을 달성한다.

### 데이터
- **WM-811K**: 172,950개 라벨링된 웨이퍼 맵, 7개 결함 클래스
- 극심한 클래스 불균형 (Donut 555개 ~ Edge-ring 9,680개)
- 학습셋: 100~6,400으로 sub-sampling하여 소량 데이터 시뮬레이션

### 방법론

**Radon 변환 (회전 불변성):**
- 원본 이미지를 각도별 투영(시노그램)으로 변환
- **회전 → Radon 특징의 이동**으로 변환됨 → CNN의 이동 불변성과 결합하면 자동으로 회전 불변 달성

**커널 뒤집기 모듈 (뒤집기 불변성):**
- 원본/뒤집힌 커널의 2 브랜치, 가중치 공유, Max-out 연산

**Multi-branch LRP**로 모델 판단 근거를 원본 공간에서 시각화

### 핵심 결과

| 모델 | 정확도 (6,400 train) |
|------|---------------------|
| Baseline CNN | 82.51 ± 3.29% |
| **Radon + Kernel Flip** | **90.84 ± 0.81%** |

소량 데이터(100개)에서도 제안 모델 74.81% vs Baseline 63.00% (12%p 우위)

### 우리 프로젝트 적용 시사점
- **radial_dist = sqrt(die_x² + die_y²)는 회전 불변 피처**: 이 논문의 Radon 변환 접근법이 이론적으로 뒷받침
- **Edge vs Center 효과**: 결함 패턴이 웨이퍼 위치에 강하게 의존 → die_x, die_y가 불량 예측에 중요
- **물리적 의미 있는 공간 피처 우선 투입**: 단순 통계보다 radial_dist, is_edge 등이 소량 데이터에서도 효과적

- 크기 기반 6개 (면적, 둘레, 장축/단축, 이심률, 견고도)

### 핵심 결과

|------|----------|-----|-----|
| Random Forest | 0.8125 | 0.7318 | 0.9759 |
| **CNN-AUG** | **0.9856** | **0.9877** | **1.0000** |

증강 전 validation accuracy ~85% → 증강 후 ~98%

### 우리 프로젝트 적용 시사점
- **밀도 기반 피처의 unit 레벨 응용**: 웨이퍼를 영역별로 분할하여 밀도 계산 → 우리도 die_x, die_y 기반으로 웨이퍼 내 영역별 WT 피처 통계량 생성 가능
- **Occlusion 분석**: 웨이퍼 **중앙 영역이 분류에 가장 중요** → radial_dist가 핵심 피처
- **클래스 불균형 대응**: 오토인코더 기반 합성 데이터 생성 → Y>0 샘플에 SMOTE-R 등 회귀용 오버샘플링 고려

## 3-3. An SVM-Based NAND Flash Endurance Prediction Method (2021)

### 논문 정보
- **저자**: Haichun Zhang 외 (화중과기대)
- **학회**: Micromachines 12(7), 746

### 연구 배경 및 목적
NAND 플래시 메모리(Micron 3D TLC)의 **내구성(endurance) 등급**을 SVM으로 예측하여, 잔여 P-E 사이클 수준과 미래 Raw Bit Error(RBE) 수준을 조기 경고한다. 동일 모델/배치 간에도 수배~10배의 내구성 편차가 존재하여 공칭값 기반 교체는 수명 낭비.

### 데이터
- Micron MT29F256G08 3D TLC NAND Flash, 96개 블록
- **5차원 입력**: P-E 사이클 수, RBE 산술평균, RBE 최대값, RBE 표준편차, 소거 시간
- **4개 독립 라벨**: 잔여 P-E 등급, 100/200/500 사이클 후 RBE 등급 (각 4-class)

### 방법론

**하위→상위 단위 집계 (핵심 유사성):**
- **페이지(page) 레벨** RBE → **블록(block) 레벨** 예측 (우리의 die→unit과 동일 패턴)
- 집계 함수: **mean**(중심 경향, 국소 간섭 완화), **max**(가장 취약 구성요소), **std**(구성요소 간 편차)

**Transient Error 최적화:**
- 5회 연속 P-R-E 사이클 평균으로 RTN 노이즈 감소
- **Pre-optimization**(노이즈 제거 먼저 → 집계 나중)이 Post보다 2.4%p 우수

**SVM(RBF커널) + OAO 다중 분류**, 5-fold CV

### 핵심 결과

| 라벨 | SVM Accuracy | DT | KNN |
|------|-------------|-----|-----|
| 잔여 P-E | **95.82%** | 90.74% | 88.08% |
| RBE Ni=200 | **93.92%** | - | - |

- Transient error 최적화로 **7.9%p 정확도 향상**
- 배드 블록 조기 경고 성공률 **96.9%**

### 우리 프로젝트 적용 시사점
- **die→unit 집계 전략의 직접 참고**: page→block ≈ die→unit. mean(중심), max(취약점), std(불균일성) 집계 근거
- **Pre-optimization 원칙**: 이상치/노이즈 처리를 die 레벨에서 먼저 수행한 후 unit 레벨로 집계하는 순서가 바람직
- **피처 간 높은 상관 처리**: RBE mean과 std의 r=0.979에도 둘 다 유용. 집계 피처 간 높은 상관 발생 시 PCA 또는 상관 쌍 제거로 대응
- **다중 등급 분류 고려**: Two-Stage Stage 1에서 이진(Y=0/Y>0) 대신 다중 등급(0, low, medium, high) 적용 가능

# 4. Tabular ML vs DL 벤치마크 (3편)

## 4-1. Tabular Data: Deep Learning is Not All You Need (2021)

### 논문 정보
- **저자**: Ravid Shwartz-Ziv, Amitai Armon (Intel IT AI Group)
- **학회**: arXiv:2106.03253v2

### 연구 배경 및 목적
TabNet, NODE, DNF-Net, 1D-CNN 등 테이블 전용 딥러닝 모델들이 "트리 모델을 능가한다"고 주장했으나, 각 논문이 서로 다른 데이터셋을 사용하여 공정한 비교가 불가. **11개 데이터셋에서 XGBoost vs 딥러닝 모델 일관된 비교** 수행.

### 실험 설계
- **11개 데이터셋** (분류+회귀, 피처 10~2,000, 샘플 7K~1M)
- **비교 모델**: XGBoost, TabNet, NODE, DNF-Net, 1D-CNN
- HyperOpt로 **모든 모델에 동일하게 1,000 step** 탐색
- 4회 반복, Friedman test (95% 신뢰수준)

### 핵심 결과

- **11개 중 8개**에서 XGBoost가 딥러닝보다 우수 (p < 0.005)
- 각 딥러닝 모델은 자기 원 논문 데이터셋에서만 강하고 **unseen 데이터에서 성능 급락** (selection bias)
- **앙상블(XGBoost + 딥러닝)이 최고 성능**: 11개 중 7개에서 유의하게 최고

**평균 상대 성능 저하율:**

| 모델 | 저하율(%) |
|------|----------|
| Deep Ensemble w/ XGBoost | **2.32** |
| XGBoost | 3.34 |
| TabNet | 10.51 |
| NODE | 14.21 |

- XGBoost는 **10배 이상 빠르게** 수렴, 기본 하이퍼파라미터에서도 이미 높은 성능

### 우리 프로젝트 적용 시사점
- **XGBoost/LightGBM을 최우선 baseline으로 사용**
- TabNet 등 딥러닝 단독 사용은 위험 (unseen 데이터에서 성능 급락)
- **최종 성능 극대화**: XGBoost + 딥러닝 앙상블로 2~3% 추가 개선 가능
- 하이퍼파라미터 탐색 예산이 제한적이면 XGBoost에 집중

## 4-2. Why Do Tree-Based Models Still Outperform Deep Learning on Typical Tabular Data? (2022)

### 논문 정보
- **저자**: Leo Grinsztajn, Edouard Oyallon, Gael Varoquaux (Inria Saclay, Sorbonne)
- **학회**: NeurIPS 2022 (Datasets and Benchmarks Track)

### 연구 배경 및 목적
트리 모델이 왜 여전히 우세한지 **근본 원인(inductive bias)**을 45개 데이터셋, 20,000 compute hours의 대규모 벤치마크로 실증 규명.

### 실험 설계
- **45개 데이터셋** (OpenML, 엄격한 선별: 이질적 컬럼, d/n < 1/10, 실제 데이터)
- **트리**: RF, GBT/HistGBT, XGBoost
- **딥러닝**: MLP, ResNet, FT-Transformer, SAINT
- 모든 모델에 **약 400회 random search**, bootstrap 분산 추정

### 핵심 결과

**성능 순위 (일관적):**
1. GradientBoostingTree / XGBoost
2. RandomForest
3. FT-Transformer (딥러닝 중 최선)
4. SAINT
5. ResNet / MLP (최하위)

**트리 모델이 우세한 3가지 근본 원인:**

**(1) 뉴럴넷은 과도하게 매끄러운(smooth) 함수를 학습**
- 실제 타겟 함수는 불규칙(irregular). 트리는 piece-wise constant로 불규칙 패턴에 강함.
- 타겟을 인위적으로 smooth하게 만들면 뉴럴넷 성능 유지, 트리 성능 하락 → 확인

**(2) 뉴럴넷(MLP)은 비정보적 피처에 취약**
- 비정보적 피처 제거 시 MLP-트리 격차 줄어듦, 추가 시 격차 벌어짐
- FT-Transformer와 트리는 상대적으로 강건

**(3) 데이터는 회전 불변이 아님**
- MLP는 회전 불변 → 테이블 데이터에서 불리 (각 피처가 고유 의미)
- FT-Transformer의 Feature Tokenizer가 회전 불변성을 깨뜨려 MLP보다 우수

### 우리 프로젝트 적용 시사점
- **우리 데이터가 트리 모델 우세 조건에 정확히 부합**:
  - 비정보적 피처 다수 (최대 target 상관 0.037)
  - 불규칙 타겟 (zero-inflated)
  - 이질적 피처 (연속형+이산형 혼합)
- **피처 선택이 딥러닝에 특히 중요**: Boruta/Null Importance로 줄인 후에야 딥러닝 시도 가치
- **FT-Transformer가 딥러닝 중 최선**: 독립 사용보다 앙상블 구성원으로 활용

1. **소규모 샘플**: 과적합 위험

3. **비공간적 관계**: CNN/RNN의 inductive bias 불작동

### 서베이 결론
- **중간 규모 데이터셋에서 트리 기반 모델이 여전히 SOTA**
- 딥러닝이 우세한 조건: 매우 큰 데이터셋, 복잡한 피처 상호작용, 다중모달, transfer learning
- 정규화 cocktail(Mixup, Dropout, Weight Decay)을 적절히 적용하면 MLP도 GBDT와 경쟁 가능

### 우리 프로젝트 적용 시사점

**데이터 특성 매칭:**

| 특성 | 트리 유리? | DL 유리? |
|------|:---------:|:--------:|
| ~44K samples | O | - |
| ~6,700 features | O | FT-Transformer 일부 |
| 피처-타겟 상관 극히 낮음 | O | X (MLP 취약) |
| Zero-inflated target | O (불규칙 함수) | X (smooth 편향) |
| 수치형+이산형 혼합 | O (네이티브) | 인코딩 필요 |

**권장 모델 우선순위:**
1. **LightGBM, XGBoost** (필수)
2. **FT-Transformer** (피처 선별 후 ~300~500개, 앙상블용)

# 5. 피처 선택 / 이상치 처리 (4편)

## 5-1. Feature Selection with the Boruta Package (2010)

### 논문 정보
- **저자**: Miron B. Kursa, Witold R. Rudnicki (University of Warsaw)
- **학회**: Journal of Statistical Software, Vol. 36, Issue 11

### 연구 배경 및 목적
기존 피처 선택은 **minimal-optimal**(최소 피처로 최고 성능)에 집중했으나, 이 논문은 **all-relevant**(유의미한 모든 피처 식별) 문제를 다룬다. 현상의 메커니즘을 이해하려면 관련된 모든 피처를 알아야 하기 때문.

### 데이터
- **Ozone**: 12개 속성 (UCI)
- **Madelon**: 500개 속성 (NIPS 2003 Feature Selection Challenge). 실제 중요 20개 + 480개 랜덤 노이즈
- 스케일링 실험: 250~2,000 객체, 500~3,500 속성 (64개 테스트셋)

### 방법론 (Boruta 알고리즘)

1. 모든 변수의 셔플된 복사본(**Shadow attribute**) 생성
2. 확장된 데이터에서 Random Forest 실행, Z-score 계산
3. Shadow 속성 중 **최대 Z-score(MZSA)** 확인
4. MZSA보다 유의하게 **높은** 속성 → "important" 확정
5. MZSA보다 유의하게 **낮은** 속성 → "unimportant" 제거
6. 모든 속성에 판정이 날 때까지 반복

**시간 복잡도**: O(P × N), 속성 수에 대해 선형. ~100만 (속성×객체)당 약 1시간.

### 핵심 결과

**Madelon**: 500개 중 정확히 **20개** important 확정, 480개 rejected (96% 감소)
- 전체 속성 사용 시 테스트 오류 ~0.28 → Boruta 선택 속성 시 **~0.10**
- 단일 RF importance보다 **더 안정적** (비중요 속성이 중요 속성보다 높은 점수 받는 경우 없음)

### 우리 프로젝트 적용 시사점
- **All-relevant 접근법의 가치**: 단일 피처-타겟 상관이 max 0.037으로 극히 약하므로, 피처 **조합**에서 나오는 약한 신호를 포착하는 Boruta가 필터 방법보다 유용
- **~6,700개에서 적용 가능**: 시간 복잡도가 선형. 다만 사전 필터링(분산/상관) 후 적용이 효율적
- Python `BorutaPy`: `RandomForestRegressor` 지원, `max_iter=200`, `n_estimators='auto'`로 설정

## 5-2. ML Feature Selection from High-Dimensional Data for Early Detection of Failures on Semiconductor Devices (2025)

### 연구 배경 및 목적

### 방법론

### 우리 프로젝트 적용 시사점

- **RFECV의 한계**: step=1로 6,700개는 비현실적. **Boruta나 LightGBM importance가 효율적**
- **다수 알고리즘 교차 검증**: RFC와 SVC가 동일 피처 선택 → 신뢰도 향상

## 5-3. Improving Electronic Sensor Reliability by Robust Outlier Screening (2013)

### 논문 정보
- **저자**: Manuel J. Moreno-Lizaranzu (Freescale Semiconductor), Federico Cuesta (U of Seville)
- **학회**: Sensors, Vol. 13, pp. 13521-13542

### 연구 배경 및 목적
자동차/의료 전자 센서의 잠재 결함을 웨이퍼 테스트 단계에서 스크리닝. **공간 알고리즘**(GDBC SB, BBBC)과 **Robust DPAT**를 Freescale 생산 환경에서 수년간 실운영.

### 데이터
- **289,080개 다이** (205,671 양품, 83,409 불량, **26개 CQI**)
- **~1,500개 테스트** 항목
- 천만 개 이상의 웨이퍼 처리 실적

### 방법론

**공간 알고리즘:**
- **GDBC** (Good Die in Bad Cluster): 이웃 중 불량 비율이 임계값 초과 시 양품도 불량 처리
- **GDBC SB** (Specific Bins): 특정 결함 유형(bin)만 고려하여 정밀도 향상
- **BBBC** (Bad Bin in Bad Cluster): 특정 bin 클러스터 주변 양품을 불량 처리

**PAT 알고리즘:**
- **DPAT**: mean ± k·σ (이상치에 편향 가능)
- **AEC DPAT**: median + k·(p99-median)·0.43 (강건)
- **NNR** (Nearest Neighbor Residual): 가우시안 가중 이웃 평균과의 잔차에 DPAT 적용. **공간 트렌드 제거**
- **R-DPAT**: Anderson-Darling 정규성 검정 → Grubbs 이상치 제거 → 비정규 시 Johnson 변환 → AEC DPAT 폴백

### 핵심 결과

**34.6% CQI 검출 시 양품 폐기율:**

| 알고리즘 | 양품 폐기율 |
|---------|-----------|
| 무작위 | 48% |
| DPAT (전체 테스트) | 40% (**무작위보다 나쁨!**) |
| NNR (특정 테스트) | 15.1% |
| **Multi-Site DPAT (특정 테스트)** | **8.9%** |

- **전체 1,500개 테스트로 DPAT 적용 시 무작위보다 나쁨** → 피처 선택 필수
- Multi-Site 분리가 가장 큰 성능 향상 (사이트 간 분산 제거)

### 우리 프로젝트 적용 시사점
- **결함 클러스터링과 공간 피처**: 실증적으로 확인 → die_x, die_y, radial_dist의 중요성 뒷받침
- **강건 이상치 처리**: 단순 mean±kσ보다 **중앙값/백분위수 기반(AEC DPAT)** 또는 **Grubbs+Johnson 변환(R-DPAT)**이 우수
- **관련 없는 피처 포함 시 성능 악화**: 1,500개 전체 → 무작위보다 나쁨. **~6,700개에서 철저한 피처 선택 필수**
- **로트별 정규화**: Multi-Site 효과처럼, run_id별 WT 피처 분포 차이를 고려한 **로트별 정규화 또는 로트 집계 통계 피처** 유효
- **NNR의 공간 잔차 아이디어**: die의 WT 값에서 인접 die 가중 평균을 뺀 **잔차(residual)**를 별도 피처로 추가 고려

## 5-4. Redefining Outliers for On-Wafer Electrical Testing (2024)

### 논문 정보
- **저자**: Michihiro Shintani (Kyoto Institute of Technology), Takashi Sato (Kyoto University)
- **학회**: MLCAD 2024 (ACM/IEEE)

### 연구 배경 및 목적
DPAT의 한계(공간 트렌드 미고려)를 **가우시안 프로세스 회귀(GPR)**로 극복. DPAT로 검출 불가능한 3가지 이상치 유형:
- **Chip A**: DPAT 한계 초과 → 검출 가능
- **Chip B**: DPAT 내이지만 주 분포에서 약간 벗어남 → 검출 불가
- **Chip C**: DPAT 내, 분포 중간이지만 **인접 칩과 크게 다름** → 절대 검출 불가

### 데이터
- Power device: SiC 웨이퍼 1장, 435개 칩
- LSI: CMOS LSI, 25개 웨이퍼, 각 ~4,000개 칩
- EGPR 실험: 40nm 자동차용 LSI, 60개 웨이퍼, 각 ~4,000개 다이. 전문가 3단계 라벨링 (alpha/beta/gamma/양품)

### 방법론

**GPR 기반 이상치 검출:**
- 칩 좌표를 입력, 측정값을 출력으로 GPR 학습 → 칩별 공간 트렌드 고려한 **적응적 임계값**
- RBF 커널: k = θ₁·exp(-(xi-xj)²/θ₂)
- **DPAT과의 차이**: DPAT는 모든 칩에 동일 임계값, GPR은 칩별 다른 임계값

**EGPR (Ensemble GPR):**
1. 데이터를 S개 배치로 분할 (Poisson Disk Sampling으로 이상치 분산)
2. 배치별 GPR 학습 (해당 배치 제외 나머지로 예측)
3. S개 예측의 **중앙값(median)**으로 앙상블 → 이상치 편향 제거
4. 측정값이 μ̃ ± Z·σ̃ 범위 밖이면 이상치

### 핵심 결과

| 방법 | Rank alpha (148) | Rank beta (58) | Rank gamma (95) |
|------|:-:|:-:|:-:|
| GPR (S=1) | 100% | 87.9% | 62.1% |
| **EGPR (S=5)** | **100%** | **96.6%** | **73.7%** |

- EGPR이 잠재 이상치(beta, gamma) 검출에서 GPR 대비 **최대 15.6% 향상**
- 계산 시간: GPR 54.1초 → EGPR(S=7) **0.76초** (98.6% 감소, GPU)

### 우리 프로젝트 적용 시사점
- **공간 트렌드의 중요성**: 동심원 패턴 등 → radial_dist, die_x, die_y 피처의 이론적 근거
- **GPR 잔차를 피처로 활용**: WT 데이터에서 각 die의 측정값과 GPR 공간 트렌드의 **잔차(residual)**를 새 피처로 추가 가능. 공간 트렌드에서 크게 벗어난 die = 불량 위험 높음
- **칩별 적응적 임계값**: 전역 IQR보다 **run_id/웨이퍼/위치별 국소 기준**이 더 효과적
- **Rank gamma (분포 내이지만 공간 이상)**: 전체 분포 기준 정상이지만 인접 die와 다른 die → die-level 공간 잔차 피처가 이를 포착
