# 1. 반도체 WT → Field 불량 예측 (9편)

## 1-1. Prediction of Field Failure Rate Using Data Mining in Automotive Semiconductor (2018)

### 논문 정보
- **저자**: Gyungsik Yun (Texas Instruments Korea), Hee-Won Jung, Sungbum Park (Hoseo University)
- **학회**: Procedia Computer Science 139, ITQM 2018

### 연구 배경 및 목적
자동차용 반도체는 수명 15년 이상, 온도 -40~150도C, 불량률 0% 목표 등 극히 높은 신뢰성 요구사항을 가진다. 기존 JEDEC/AEC 표준 기반 가속 수명 시험은 고객의 다양한 사용 조건을 반영하지 못한다. 반도체 조립(assembly) 공정과 패키지 테스트(package test) 데이터를 데이터마이닝으로 분석하여 **잠재적 Field 불량을 예측**하는 모델을 제안한다.

### 데이터
- 외국계 반도체 회사(A사)의 3년간 실생산 데이터
- **86개 품질 지표** (조립 공정 + 패키지 테스트)
- 총 **1,307개 Lot**: 불량 발생 132개, 미발생 1,175개 (불균형 비율 8.9:1)
- 2-class 분류 문제 (Field 불량 발생/미발생)

### 방법론
- 트리 기반 알고리즘 4종 비교: Decision Tree, **Random Forest**(50개 트리), ExtraTrees, AdaBoost
- 3-Fold Cross-Validation
- RF Feature Importance로 상위 9개 핵심 변수 선별 후 2개씩 쌍으로 추출하여 Decision Boundary 시각화

### 핵심 결과

| 모델 | 학습 정확도 | 테스트 정확도 |
|------|-----------|-------------|
| Random Forest | 100% | **92.8%** |
| Zero-R (다수 클래스 예측) | - | 89.7% |

**Feature Importance Top 9**: 패키지 테스트 공정 변수 6개 + 조립 공정 변수 3개. 조립 공정 불량률이 높은 Lot은 열적/물리적 스트레스가 상대적으로 높아 Field 불량 확률 증가. **ExtraTrees가 모든 feature 쌍에서 최고 성능**.

### 우리 프로젝트 적용 시사점
- **문제 구조 동일**: 패키지 테스트/조립 데이터 → Field 불량 예측 ≈ WT 데이터 → RCC 예측
- **ExtraTrees의 우수성**: baseline 모델 비교에 ExtraTrees 포함 타당성 확인
- **Feature Importance 기반 선별** 필수: 86개에서 9개 핵심 변수만으로 충분
- 결론에서 "회귀 확장 필요" 언급 → Two-Stage(분류+회귀) 접근법 타당성 확인

## 1-2. Applying ML Models on Metrology Data for Predicting Device Electrical Performance (2023)

### 논문 정보
- **저자**: Bappaditya Dey 외 (imec, Heriot-Watt University)
- **학회**: arXiv:2312.09462

### 연구 배경 및 목적
3nm 이하 반도체 공정에서 Overlay 오차가 최종 전기적 성능(정전용량)에 미치는 영향을 정량화하고, 초기 공정 단계의 계측 데이터로 **최종 전기적 성능을 ML로 조기 예측**하여 불량 웨이퍼의 rework/scrap 조기 결정을 지원한다.

### 데이터
- imec N-14 BEOL 공정, 4장 웨이퍼 (2장 Non-Programmed + 2장 Programmed Overlay)
- CD 예측: **30개 입력 피처** (26개 overlay + die 위치 X,Y + 타겟 위치 X,Y), 웨이퍼당 700+ 데이터포인트
- 정전용량 예측: **29개 입력 피처**, 80:20 train/test 분할

### 방법론
- 4종 ML 회귀 모델 비교: Linear Regression, SVR(RBF), Random Forest, **Extra Tree Regressor**
- 정규화(단위 분산 스케일링) 적용
- **DBSCAN** 알고리즘으로 타겟 변수 이상치 탐지 및 클리닝

### 핵심 결과

**CD 예측 (AEI DBO, AB1):**

| 모델 | R² (test) | MSE |
|------|----------|-----|
| Linear Regression | 0.836 | 1.168 |
| SVR | 0.853 | 1.045 |
| Random Forest | 0.876 | 0.881 |
| **Extra Tree Regressor** | **0.891** | **0.779** |

**정전용량 예측**: DBSCAN 적용 후 R² 0.742 → **0.950**, MSE 182.100 → **0.077** (극적 개선)

### 우리 프로젝트 적용 시사점
- **Extra Tree Regressor가 모든 실험에서 RF보다 우수** → baseline에 ExtraTrees 반드시 포함
- **DBSCAN 기반 이상치 클리닝**: health 데이터의 max=1.0 등 극단값에 DBSCAN 탐지 고려
- **Die 위치 피처(X, Y)가 성능 향상에 기여** → run_wf_xy에서 die_x, die_y 파싱 (Stage 4.5)

## 1-3. Rare Class Prediction Model for Smart Industry in Semiconductor Manufacturing (2024)

### 논문 정보
- **저자**: Abdelrahman Farrag, Mohammed-Khalil Ghali, Yu Jin (SUNY Binghamton)
- **학회**: arXiv:2406.04533

### 연구 배경 및 목적
반도체 제조 센서 데이터의 **심각한 클래스 불균형**(1:14), **높은 결측률**(4.5%), **노이즈 피처**(591개) 세 가지 난제를 동시에 해결하여 희소 클래스(불량) 예측 정확도를 높이는 구조화된 접근법 개발.

### 데이터
- **SECOM 데이터셋** (UCI ML Repository): 1,567개 샘플 × 591개 센서 측정값
- 불량 샘플 104개 (클래스 불균형 1:14), 결측률 4.5%

### 방법론

**전처리 순서:**
1. 결측률 50% 초과 피처 제거 → 28개 삭제
2. 상수값 피처 제거 → 116개 제거
3. k-NN imputation (최고 클래스 분리 달성)
4. 상관계수 70% 초과 피처 제거 → **204개**로 축소

**투표 기반 피처 선택 (핵심 기여):**
12가지 피처 선택 알고리즘(ANOVA, Mutual Information, **Boruta**, MARS, RFE by LR/SVM/RF, LASSO, Sequential FS by XGB/SVM 등)의 투표로 **최소 3개 이상에서 선택된 81개 피처** 최종 채택

**리샘플링**: SMOTE oversampling(소수 70%) + undersampling(다수 80%) 결합이 최적

**모델**: LR, SVM, DTC, RF, GBC, **XGBoost** 6종 비교

### 핵심 결과

| 시나리오 | 모델 | AUC | Recall | Precision | FAR |
|---------|------|-----|--------|-----------|-----|
| Combined (최적) | XGBoost | **0.95** | **0.96** | 0.66 | **0.03** |

기존 연구(Deep NN+PSO: Recall 0.98, In-painting KNN: 0.78) 대비 Recall과 AUC에서 개선, FAR 최저.

### 우리 프로젝트 적용 시사점
- **투표 기반 피처 선택**: Boruta + LightGBM importance + Null Importance + Permutation Importance 결과를 교차 검증(투표식)하여 최종 피처 선정
- **전처리 순서**: 결측률 50% 초과 제거 → 상수값 제거 → k-NN imputation → Stage 1~2 전략과 일치
- **SMOTE + undersampling 결합**: Two-Stage 모델의 Stage 1(Y=0 vs Y>0 분류) 학습 시 적용
- **XGBoost의 일관된 우수성**: 불균형 데이터에서 XGBoost가 최고 → Stage 1 분류기에 사용 근거

## 1-4. Yield Prediction in Semiconductor Manufacturing Using Two-Step Machine Learning (2025)

### 논문 정보
- **저자**: Rebecca Busch 외 (University of Siegen, Elmos Semiconductor SE)
- **학회**: International Journal of Production Research, Taylor & Francis

### 연구 배경 및 목적
**FDC(Fault Detection and Classification) 데이터**만으로 생산 초기 단계에서 최종 전기적 수율을 예측하는 최초의 시도. **이상치(불량 웨이퍼)를 먼저 분류하여 제거한 뒤 양품만으로 수율 예측**하는 Two-Step 접근법 제안.

### 데이터
- 주요 파운드리(Elmos Semiconductor SE)의 실생산 데이터, 350nm 노드
- **21,630개 웨이퍼**, 6개월간 수집, **1,188개 → 전처리 후 804개 파라미터**
- 학습 15,140개 / 테스트 6,490개

### 방법론

**Boruta(WFS) 피처 선택**: 804개 → **76개** 파라미터로 축소 (비교: 상관 필터링 352개, PCA는 성능 열위)

**Two-Step 모델:**
- **Step 1 (분류)**: Random Forest로 3개 클래스 분류 (acceptable/average/poor). 정확도 **0.990**.
- **Step 2 (회귀)**: acceptable로 분류된 데이터만으로 수율 예측

**5종 ML 비교**: RF, Neural Networks, KNN, GLM, GLMBoost

### 핵심 결과

| 모델 | 테스트 데이터 | R² | RMSE | MAE |
|------|------------|-----|------|-----|
| m1 (전체, 804 params) | 전체 | 0.192 | 0.031 | 0.012 |
| m2 (acceptable, 804) | 실제 acceptable | **0.496** | 0.130 | 0.009 |
| m4 (acceptable + WFS, 76) | **실제 acceptable** | **0.538** | **0.011** | **0.008** |

- **Two-Step**: R² 0.192 → **0.496** (전체 대비 실제 acceptable)
- **Boruta 추가**: R² 0.496 → **0.538** (804→76개에서 오히려 향상)
- **RF가 모든 조건에서 최고 성능**

### 우리 프로젝트 적용 시사점
- **Two-Stage 전략의 직접적 근거**: 분류→회귀 구조가 R²를 0.192→0.538로 극적 향상. 우리의 Y=0 vs Y>0 분류 → Y>0 회귀 전략과 동일
- **Boruta의 피처 선택 효과**: 804→76개로 축소하면서 R²가 오히려 향상. 노이즈 피처 제거가 핵심
- **분류 정확도의 중요성**: 1% 미만의 분류 오류도 회귀 성능을 크게 저하 → Stage 1 분류기 최적화 필수
- **PCA는 부적합**: 해석 불가능하고 성능 열위. Boruta 등 해석 가능한 피처 선택 방법이 적합

## 1-5. A Machine Learning-based Approach for Failure Prediction at Cell Level based on WAT Parameters (2021)

> *웹 공개 정보 기반 요약*

### 논문 정보
- **저자**: Xiang Chen, Yi Zhao, Hongliang Lü, Xiaoqiang Shao, Cheng Chen, Yu Huang (Huawei Technologies)
- **학회**: IEEE Microelectronics Design & Test Symposium (MDTS)
- **URL**: https://ieeexplore.ieee.org/document/9476151

### 연구 배경 및 목적
WAT(Wafer Acceptance Test) 데이터와 후속 진단 보고서(diagnosis report) 간의 관계가 충분히 탐색되지 않았다. Huawei의 산업용 WAT 데이터셋을 사용하여 **셀 레벨에서의 불량 예측 프레임워크**를 제안한다.

### 방법론
- **피처 선택**: RFECV(Recursive Feature Elimination with Cross Validation), Pearson 상관 분석
- **클래스 불균형 처리**: SMOTE (Synthetic Minority Oversampling Technique)
- **모델**: XGBoost 등 ML 모델 비교
- 학습 프레임워크에 피처 선택 + 불균형 처리를 통합

### 핵심 결과
- Accuracy **0.95+**, F1-score **0.95+**, AUROC **0.95+** 달성
- 산업용 실데이터에서 검증

### 우리 프로젝트 적용 시사점
- **WAT → 불량 예측 구조가 우리와 가장 유사**: WT 데이터에서 셀/die 레벨 불량을 예측하는 프레임워크
- **RFECV + SMOTE + XGBoost** 파이프라인이 산업 데이터에서 효과적임을 확인
- Huawei 산업 데이터에서 0.95+ 성능은, 우리 SK Hynix 데이터에서도 WT가 Field Health에 대한 예측력을 가진다는 간접 근거

## 1-6. Using Wafer Map Features to Better Predict Die-Level Failures in Final Test (2015)

> *웹 공개 정보 기반 요약 | **SK Hynix 공저 — 우리 프로젝트에 가장 직접적으로 관련된 논문***

### 논문 정보
- **저자**: Seokho Kang, Sungzoon Cho (서울대), **Daewoong An, Jaeyoung Rim (SK Hynix DRAM Development Division)**
- **학회**: IEEE Transactions on Semiconductor Manufacturing, Vol. 28, No. 3, pp. 431-437
- **URL**: https://ieeexplore.ieee.org/document/7120993

### 연구 배경 및 목적
불량 반도체 die가 wafer 레벨 테스트를 통과하고 최종 조립 테스트에서 실패하는 문제를 해결. **웨이퍼 맵에서 도출한 공간 피처**로 이러한 잠재 불량을 사전 예측.

### 방법론 (핵심)
**4가지 웨이퍼 맵 파생 변수:**
1. **웨이퍼 중심으로부터의 거리 (distance from wafer center)** — edge effect 포착
2. **해당 die 위치의 과거 최종 수율 (historical final yield at die position)** — 위치별 불량 이력
3. **인접 die의 wafer test 불량률 (failure rate among adjacent dies)** — 클러스터링 효과
4. **웨이퍼 맵 내 이상 패턴 (abnormal wafer map patterns)** — 전체 맵 수준 이상 감지

**모델**: Random Forest 분류기

### 핵심 결과
- 4가지 파생 변수 사용 시 기존 WT 피처만 사용한 경우 대비 **통계적으로 유의한 예측 성능 향상**
- SK Hynix DRAM 실생산 데이터에서 검증

### 우리 프로젝트 적용 시사점

| 파생 변수 | 우리 프로젝트 대응 |
|-----------|------------------|
| 웨이퍼 중심 거리 | `radial_dist = sqrt(die_x² + die_y²)` (Stage 4.5) |
| die 위치별 과거 수율 | `run_id`별 해당 위치의 WT 통계 (로트 집계 피처) |
| 인접 die 불량률 | 같은 unit 내 다른 die의 WT 값 (position별 집계) |
| 웨이퍼 맵 이상 패턴 | 로트/웨이퍼 단위 이상 탐지 피처 |

- **CLAUDE.md Stage 4.5 전략의 학술적 근거**: die_x, die_y, radial_dist, is_edge 피처가 정확히 이 논문의 변수 1에 대응
- **인접 die 정보**: unit 내 4개 die의 WT 피처를 std, range로 집계하면 변수 3의 효과를 간접 포착
- SK Hynix 데이터에서 검증된 결과이므로 **우리 데이터에도 직접 적용 가능성 높음**

## 1-7. A Wafer Map Yield Prediction Based on Machine Learning for Productivity Enhancement (2019)

> *웹 공개 정보 기반 요약*

### 논문 정보
- **저자**: Sung-Ju Jang, Jong-Seong Kim, Tae-Woo Kim, Hyun-Jin Lee, Seungbum Ko (Samsung Electronics)
- **학회**: IEEE Transactions on Semiconductor Manufacturing, Vol. 32, No. 4, pp. 400-407

## 2-1. Hybrid ML Approach to Zero-Inflated Data Improves Accuracy of Dengue Prediction (2024)

### 논문 정보
- **저자**: Micarnaldo Ernesto Francisco, Thaddeus M. Carvajal, Kozo Watanabe 외
- **학회**: PLOS Neglected Tropical Diseases

### 연구 배경 및 목적
뎅기열 발생률 예측에서 fine spatiotemporal resolution(주간-마을 단위)의 데이터는 **72% 이상이 0**이어서 ML 모델이 0을 과도하게 예측한다. 기존 hurdle/ZI 모델이 이산(count) 데이터에 한정된 반면, **연속 데이터에 적용 가능한 Hybrid(분류+회귀) 접근법**을 제안한다.

### 데이터
- 필리핀 Metro Manila, 1,806개 마을, 2009~2013 (train), 2014 (test)
- **종속변수**: 주간 뎅기 발생률 (연속 변수, log(n+1) 변환)

- **Zero 비율**: training 71.4%, test 65.8%

### 방법론

**6종 ML 비교**: GAM, RF, CIF(Conditional Inference Forest), ANN, SVR, XGB

**Hybrid 2단계 접근법 (핵심):**
1. **Stage 1 (Qualitative/분류)**: **RF**로 뎅기 발생 여부 예측 (0 vs 1). AUC=**0.899**
2. **Stage 2 (Quantitative/회귀)**: "발생(1)"으로 예측된 위치-시점만 **CIF**로 정량 예측
3. Stage 1에서 0 예측 → 최종 0, Stage 1에서 1 예측 → Stage 2 회귀값 출력

**HAI (Hybrid Accuracy Index)**: AUC × 0.7 + S-R²ₐ × 0.3 (zero 비율에 비례한 가중치)

### 핵심 결과

| 접근법 | 정량 예측 R² | 비고 |
|--------|------------|------|
| Quantitative 단독 (CIF) | 0.117 (S-R²ₐ) | zero-inflation으로 인해 극히 낮음 |
| Hybrid (RF분류 + CIF회귀) | **~0.716** | Stage 1에서 0 필터링 후 R² 대폭 개선 |

- **Aggregation으로 zero 비율 감소**: 공간 해상도를 낮추면(village→city) zero가 줄어 정량 모델 정확도 향상

### 우리 프로젝트 적용 시사점
- **Two-Stage 구조 직접 적용 가능**: 우리 health data도 70.8%가 0이고 **연속값**이므로 이 논문의 hybrid 접근법이 정확히 동일 구조
  - Stage 1: Classifier로 Y=0 vs Y>0
  - Stage 2: Y>0 예측된 샘플만 Regressor로 health 값 예측
  - 최종: P(Y>0) × E[Y|Y>0]
- **각 Stage에 다른 알고리즘 배치**: 1단계(RF) ≠ 2단계(CIF). 우리도 Stage 1에 LightGBM, Stage 2에 XGBoost 등 조합 실험 필요
- **연속 데이터에 Hybrid 적용 가능 확인**: 기존 hurdle/ZI 모델은 count 전용이지만, hybrid 방식은 연속값에도 적용 가능

## 2-2. An Interpretable ML-Based Hurdle Model for Zero-Inflated Data Analysis (2024)

### 논문 정보
- **저자**: Moataz Bellah Ben Khedher, Dukgeun Yun
- **학회**: Applied Sciences (MDPI), Vol. 14, No. 23

### 연구 배경 및 목적
교통사고 빈도 데이터(81% zeros, overdispersion)에서 **CatBoost 기반 ML Hurdle 모델**을 개발하고, **Custom Loss Function**으로 class imbalance 대응, **SHAP 분석**으로 해석 가능성 확보.

### 데이터
- 한국도로공사(KICT), 2011~2015년 일반국도
- **8,636개 도로 구간**, 총 3,182건 사고
- **Zero 비율**: 81.7% (7,052 구간이 사고 0건)
- 독립변수: 구간 길이, 경사도, 곡률, 차로폭, AADT 등

### 방법론

**수학적 정의:**
```
최종 예측: E[Y|X] = phi_hat(x) × mu_hat(x)
  phi_hat(x) = P(Y>0)  (분류 모델 출력)
  mu_hat(x) = E[Y|Y>0]  (회귀 모델 출력)
```

**Stage 1 (CatBoost Classifier):**
- **Custom Weighted Log-Loss**: `w_Cj = 1/n_Cj` (클래스 빈도 역수)로 소수 클래스에 높은 가중치

**Stage 2 (CatBoost Regressor):**
- **Custom Poisson Loss**: `L = -Σ(yi·log(ŷi) - ŷi)` → count/비음수 데이터 특성에 적합
- Y>0 데이터만으로 학습 (zero-truncated)

### 핵심 결과

**Classification Stage에서 Custom Loss 효과:**

| 모델 | Recall | F1 |
|------|--------|-----|
| CatBoost (Default) | 0.26 | 0.35 |
| **CatBoost (Custom Loss)** | **0.77** | **0.78** |

**Full Model 비교:**

| 모델 | RMSE | MAE |
|------|------|-----|
| Poisson Hurdle | 2.097 | 0.527 |
| NB Hurdle | 1.336 | 0.511 |
| CatBoost (Single Stage) | 1.197 | 0.421 |
| **CatBoost Hurdle (Proposed)** | **0.978** | **0.346** |

- Single-stage 대비 RMSE **18.3% 감소**, NB Hurdle 대비 **26.8% 감소**

### 우리 프로젝트 적용 시사점
- **Custom Loss Function의 중요성**: Classification에서 weighted log-loss로 recall 0.26→0.77 달성. 우리도 Stage 1에서 class_weight 또는 custom loss 적용 필수
- **Regression에서 Poisson Loss > MSE**: 우리도 Stage 2에서 Poisson/Tweedie loss 실험 필요
- **SHAP 기반 Stage별 해석**: 어떤 WT feature가 "불량 발생 여부"(Stage 1)와 "불량 규모"(Stage 2)에 기여하는지 분리 해석 가능 → 공정 개선 인사이트 도출
- **Single-stage vs Two-stage**: 18.3% RMSE 개선 → Stage 7-B(Two-Stage Model)의 기대 효과 정량적 뒷받침

## 2-3. Yield Prediction via Spatial Modeling of Clustered Defect Counts Across a Wafer Map (2007)

### 논문 정보
- **저자**: Suk Joo Bae (한양대), Jung Yoon Hwang (삼성전자), Way Kuo (U of Tennessee)
- **학회**: IIE Transactions, Vol. 39, No. 12

### 연구 배경 및 목적
웨이퍼 결함은 공간적으로 클러스터링되며 zero-defect chip이 과다하다. 기존 Poisson(균일 분포 가정→수율 과소추정)과 Negative Binomial(공간 패턴 무시)의 한계를 극복하기 위해, IC 칩의 **공간 좌표(radial, angular)를 공변량**으로 사용하는 **Zero-Inflated Poisson(ZIP) regression** 제안.

### 데이터
- 20×20 = 400개 IC 칩 (하나의 웨이퍼)
- **종속변수**: 각 칩의 결함 수 (0~12개, count data)
- **Zero 비율**: 318/400 = **79.5%** → zero-inflated
- **독립변수**: 극좌표 {r, cos(φ), sin(φ), r·cos(φ), r·sin(φ)}

### 방법론

**ZIP 모델의 두 가지 zero 소스:**
```
P(N=0) = π + (1-π)·exp(-Λ)     [구조적 zero + 우연적 zero]
P(N=k) = (1-π)·exp(-Λ)·Λᵏ/k!   (k≥1)
```
- **π (구조적 zero)**: 결함 발생이 불가능한 완벽 상태의 칩 확률
- **(1-π)·exp(-Λ) (우연적 zero)**: 결함 가능하나 우연히 0

**두 개의 regression:**
```
log(Λ) = 1.4627 - 0.0716·r + 0.0986·cos(φ)     [Poisson part: 결함 수]
logit(p) = 1.3094 - 0.2764·sin(φ) + 0.1044·r·sin(φ)  [Zero part: 완벽 확률]
```

### 핵심 결과

| 모델 | 예측 수율 | 관측 수율 | 오차 |
|------|----------|----------|------|
| Poisson (공간X) | 52.33% | 79.5% | -27.17% |
| NB (공간O) | 79.32% | 79.5% | -0.18% |
| **ZIP (공간O)** | **79.53%** | 79.5% | **+0.03%** |

- **ZIP이 가장 정확** (오차 0.03%), overdispersion도 1.620으로 NB(7.546)보다 낮음

### 우리 프로젝트 적용 시사점
- **공간 피처의 결정적 중요성**: 공간 정보 없는 Poisson(오차 27%) vs ZIP+공간(오차 0.03%). Stage 4.5의 radial_dist, die_x, die_y가 유사 역할
- **두 가지 zero 소스 개념**: Y=0 중 "진짜 양품"과 "잠재 불량"의 구분 → Two-Stage의 Stage 1이 이 구분을 학습
- **피처 엔지니어링 + 모델 구조의 시너지**: 공간 피처 + zero 분리 결합이 압도적 성능 차이를 만듦

## 2-4. Zero-Inflated Poisson Regression, with an Application to Defects in Manufacturing (1992)

> *웹 공개 정보 기반 요약*

### 논문 정보
- **저자**: Diane Lambert (AT&T Bell Laboratories)
- **학회**: Technometrics, Vol. 34, No. 1, pp. 1-14 (인용 3,856+회)
- **URL**: https://www.tandfonline.com/doi/abs/10.1080/00401706.1992.10485228

### 연구 배경 및 목적
제조 공정에서 결함 수(defect count) 데이터는 **excess zeros**(과도한 0값)가 발생한다. 표준 Poisson 회귀로는 이를 적절히 모델링할 수 없다. **Zero-Inflated Poisson(ZIP) 회귀** 모델을 제안하여, 0이 발생하는 두 가지 메커니즘을 분리 모델링한다.

### 방법론 (ZIP 모델의 수학적 정의)

**핵심 아이디어**: 데이터의 0은 두 가지 소스에서 발생한다:
1. **구조적 zero (structural zero)**: 시스템이 "완벽한 상태(perfect state)"에 있어 결함이 발생할 수 없는 경우
2. **우연적 zero (sampling zero)**: 시스템이 "불완벽한 상태"이나 우연히 결함이 0인 경우

**확률 모델:**
```
P(Y=0) = π + (1-π) · exp(-λ)          [구조적 zero + 우연적 zero]
P(Y=k) = (1-π) · exp(-λ) · λᵏ / k!    (k = 1, 2, 3, ...)
```

여기서:
- `π` = P(완벽한 상태) — 결함이 원천적으로 불가능한 확률
- `λ` = 불완벽한 상태에서의 기대 결함 수

**회귀 구조 (두 개의 동시 모델):**
```
logit(π) = z'γ     [완벽 상태 여부를 설명하는 공변량 z와 계수 γ]
log(λ)   = x'β     [결함 수를 설명하는 공변량 x와 계수 β]
```

**추정 방법**: EM(Expectation-Maximization) 알고리즘
- E-step: 각 관측치가 "완벽 상태"에서 왔을 사후 확률 계산
- M-step: 가중 GLM으로 γ와 β를 갱신

**제조업 적용 사례**: 인쇄 회로 기판(PCB) 솔더링 결함
- 장비 정렬(alignment)이 잘 되면 → 완벽 상태 (π, 결함 0 보장)
- 장비 정렬이 불완전하면 → 불완벽 상태 (Poisson 분포로 결함 발생)

### 핵심 결과
- 표준 Poisson보다 **overdispersion을 크게 감소**시킴
- 0의 과다를 명시적으로 모델링하여 비-zero 부분의 예측 정확도 향상
- 3,856+회 인용으로 zero-inflated 모델링의 표준 참고문헌으로 정착

### 우리 프로젝트 적용 시사점

| 개념 | 우리 프로젝트 대응 |
|------|------------------|
| **구조적 zero** (완벽한 상태) | Y=0인 unit 중 "진짜 양품" — Field에서 불량이 발생할 수 없는 unit |
| **우연적 zero** (우연히 0) | Y=0인 unit 중 "잠재 불량" — 아직 드러나지 않았지만 불량 위험이 있는 unit |
| **두 모델의 분리** | **Two-Stage의 이론적 근거**: Stage 1(분류)이 π를 학습, Stage 2(회귀)가 λ를 학습 |
| **EM 알고리즘** | Stage 1의 확률 출력 P(Y>0)을 Stage 2의 가중치로 사용하는 soft 접근법도 가능 |

- **우리 데이터의 Y=0 (70.8%)이 모두 동질적이지 않다**: 일부는 진짜 양품, 일부는 잠재 불량. Two-Stage 모델이 이 구분을 학습
- Lambert(1992)의 count 데이터 한정 모델을 **연속 데이터에 확장한 것이 2-1(Hybrid ML)**

- **학회**: Scientific Reports (Nature) 13:8127

### 연구 배경 및 목적
웨이퍼 맵 결함 패턴 분류에서 기존 CNN은 **회전/뒤집기 불변성**을 갖추지 못해 데이터 증강에 의존한다. **Radon 변환 + 커널 뒤집기(Kernel Flip)**를 결합하여 모델 아키텍처 자체에 기하학적 불변성을 내장, 소량 데이터에서도 높은 성능을 달성한다.

### 데이터
- **WM-811K**: 172,950개 라벨링된 웨이퍼 맵, 7개 결함 클래스
- 극심한 클래스 불균형 (Donut 555개 ~ Edge-ring 9,680개)
- 학습셋: 100~6,400으로 sub-sampling하여 소량 데이터 시뮬레이션

### 방법론

**Radon 변환 (회전 불변성):**
- 원본 이미지를 각도별 투영(시노그램)으로 변환
- **회전 → Radon 특징의 이동**으로 변환됨 → CNN의 이동 불변성과 결합하면 자동으로 회전 불변 달성

**커널 뒤집기 모듈 (뒤집기 불변성):**
- 원본/뒤집힌 커널의 2 브랜치, 가중치 공유, Max-out 연산

**Multi-branch LRP**로 모델 판단 근거를 원본 공간에서 시각화

### 핵심 결과

| 모델 | 정확도 (6,400 train) |
|------|---------------------|
| Baseline CNN | 82.51 ± 3.29% |
| **Radon + Kernel Flip** | **90.84 ± 0.81%** |

소량 데이터(100개)에서도 제안 모델 74.81% vs Baseline 63.00% (12%p 우위)

### 우리 프로젝트 적용 시사점
- **radial_dist = sqrt(die_x² + die_y²)는 회전 불변 피처**: 이 논문의 Radon 변환 접근법이 이론적으로 뒷받침
- **Edge vs Center 효과**: 결함 패턴이 웨이퍼 위치에 강하게 의존 → die_x, die_y가 불량 예측에 중요
- **물리적 의미 있는 공간 피처 우선 투입**: 단순 통계보다 radial_dist, is_edge 등이 소량 데이터에서도 효과적

## 3-2. Wafer Map Defect Classification Using Autoencoder-Based Data Augmentation and CNN (2024)

### 논문 정보
- **저자**: Yin-Yin Bao 외 (Lanzhou University of Technology)
- **학회**: arXiv:2411.11029

### 연구 배경 및 목적
웨이퍼 결함 맵 분류에서 **노이즈**, **클래스 불균형**(Edge-Ring 9,680 vs Scratch 149), **라벨링 부족**(전체의 3.1%만 결함)을 해결하기 위해 **오토인코더 기반 데이터 증강 + CNN** 결합 모델 제안.

### 데이터
- **WM-811K**: 811,457개 웨이퍼 중 25,519개 결함 데이터 (8개 클래스)
- 증강 후 각 클래스 10,000개로 균등화 (총 80,000개)

### 방법론

**오토인코더 증강:**
1. 인코더로 잠재 공간 압축 (26×26×3 → 13×13×64)
2. 가우시안 노이즈 주입: Z_noisy = Z + ε, ε ~ N(0, σ²)
3. 디코더로 복원 → 합성 웨이퍼 맵 생성

**59개 hand-crafted 피처** (전통 ML 비교용):
- 밀도 기반 13개 (웨이퍼를 9개 중심 + 4개 가장자리 영역 분할)
- Radon 변환 기반 40개 (방향별 투영의 mean, std)
- 크기 기반 6개 (면적, 둘레, 장축/단축, 이심률, 견고도)

### 핵심 결과

| 모델 | Accuracy | F1 | AUC |
|------|----------|-----|-----|
| Random Forest | 0.8125 | 0.7318 | 0.9759 |
| **CNN-AUG** | **0.9856** | **0.9877** | **1.0000** |

증강 전 validation accuracy ~85% → 증강 후 ~98%

### 우리 프로젝트 적용 시사점
- **밀도 기반 피처의 unit 레벨 응용**: 웨이퍼를 영역별로 분할하여 밀도 계산 → 우리도 die_x, die_y 기반으로 웨이퍼 내 영역별 WT 피처 통계량 생성 가능
- **Occlusion 분석**: 웨이퍼 **중앙 영역이 분류에 가장 중요** → radial_dist가 핵심 피처
- **클래스 불균형 대응**: 오토인코더 기반 합성 데이터 생성 → Y>0 샘플에 SMOTE-R 등 회귀용 오버샘플링 고려

## 3-3. An SVM-Based NAND Flash Endurance Prediction Method (2021)

### 논문 정보
- **저자**: Haichun Zhang 외 (화중과기대)
- **학회**: Micromachines 12(7), 746

### 연구 배경 및 목적
NAND 플래시 메모리(Micron 3D TLC)의 **내구성(endurance) 등급**을 SVM으로 예측하여, 잔여 P-E 사이클 수준과 미래 Raw Bit Error(RBE) 수준을 조기 경고한다. 동일 모델/배치 간에도 수배~10배의 내구성 편차가 존재하여 공칭값 기반 교체는 수명 낭비.

### 데이터
- Micron MT29F256G08 3D TLC NAND Flash, 96개 블록
- **5차원 입력**: P-E 사이클 수, RBE 산술평균, RBE 최대값, RBE 표준편차, 소거 시간
- **4개 독립 라벨**: 잔여 P-E 등급, 100/200/500 사이클 후 RBE 등급 (각 4-class)

### 방법론

**하위→상위 단위 집계 (핵심 유사성):**
- **페이지(page) 레벨** RBE → **블록(block) 레벨** 예측 (우리의 die→unit과 동일 패턴)
- 집계 함수: **mean**(중심 경향, 국소 간섭 완화), **max**(가장 취약 구성요소), **std**(구성요소 간 편차)

**Transient Error 최적화:**
- 5회 연속 P-R-E 사이클 평균으로 RTN 노이즈 감소
- **Pre-optimization**(노이즈 제거 먼저 → 집계 나중)이 Post보다 2.4%p 우수

**SVM(RBF커널) + OAO 다중 분류**, 5-fold CV

### 핵심 결과

| 라벨 | SVM Accuracy | DT | KNN |
|------|-------------|-----|-----|
| 잔여 P-E | **95.82%** | 90.74% | 88.08% |
| RBE Ni=200 | **93.92%** | - | - |

- Transient error 최적화로 **7.9%p 정확도 향상**
- 배드 블록 조기 경고 성공률 **96.9%**

### 우리 프로젝트 적용 시사점
- **die→unit 집계 전략의 직접 참고**: page→block ≈ die→unit. mean(중심), max(취약점), std(불균일성) 집계 근거
- **Pre-optimization 원칙**: 이상치/노이즈 처리를 die 레벨에서 먼저 수행한 후 unit 레벨로 집계하는 순서가 바람직
- **피처 간 높은 상관 처리**: RBE mean과 std의 r=0.979에도 둘 다 유용. 집계 피처 간 높은 상관 발생 시 PCA 또는 상관 쌍 제거로 대응
- **다중 등급 분류 고려**: Two-Stage Stage 1에서 이진(Y=0/Y>0) 대신 다중 등급(0, low, medium, high) 적용 가능

# 4. Tabular ML vs DL 벤치마크 (4편)

## 4-1. Tabular Data: Deep Learning is Not All You Need (2021)

### 논문 정보
- **저자**: Ravid Shwartz-Ziv, Amitai Armon (Intel IT AI Group)
- **학회**: arXiv:2106.03253v2

### 연구 배경 및 목적
TabNet, NODE, DNF-Net, 1D-CNN 등 테이블 전용 딥러닝 모델들이 "트리 모델을 능가한다"고 주장했으나, 각 논문이 서로 다른 데이터셋을 사용하여 공정한 비교가 불가. **11개 데이터셋에서 XGBoost vs 딥러닝 모델 일관된 비교** 수행.

### 실험 설계
- **11개 데이터셋** (분류+회귀, 피처 10~2,000, 샘플 7K~1M)
- **비교 모델**: XGBoost, TabNet, NODE, DNF-Net, 1D-CNN
- HyperOpt로 **모든 모델에 동일하게 1,000 step** 탐색
- 4회 반복, Friedman test (95% 신뢰수준)

### 핵심 결과

- **11개 중 8개**에서 XGBoost가 딥러닝보다 우수 (p < 0.005)
- 각 딥러닝 모델은 자기 원 논문 데이터셋에서만 강하고 **unseen 데이터에서 성능 급락** (selection bias)
- **앙상블(XGBoost + 딥러닝)이 최고 성능**: 11개 중 7개에서 유의하게 최고

**평균 상대 성능 저하율:**

| 모델 | 저하율(%) |
|------|----------|
| Deep Ensemble w/ XGBoost | **2.32** |
| XGBoost | 3.34 |
| TabNet | 10.51 |
| NODE | 14.21 |

- XGBoost는 **10배 이상 빠르게** 수렴, 기본 하이퍼파라미터에서도 이미 높은 성능

### 우리 프로젝트 적용 시사점
- **XGBoost/LightGBM을 최우선 baseline으로 사용**
- TabNet 등 딥러닝 단독 사용은 위험 (unseen 데이터에서 성능 급락)
- **최종 성능 극대화**: XGBoost + 딥러닝 앙상블로 2~3% 추가 개선 가능
- 하이퍼파라미터 탐색 예산이 제한적이면 XGBoost에 집중

## 4-2. Why Do Tree-Based Models Still Outperform Deep Learning on Typical Tabular Data? (2022)

### 논문 정보
- **저자**: Leo Grinsztajn, Edouard Oyallon, Gael Varoquaux (Inria Saclay, Sorbonne)
- **학회**: NeurIPS 2022 (Datasets and Benchmarks Track)

### 연구 배경 및 목적
트리 모델이 왜 여전히 우세한지 **근본 원인(inductive bias)**을 45개 데이터셋, 20,000 compute hours의 대규모 벤치마크로 실증 규명.

### 실험 설계
- **45개 데이터셋** (OpenML, 엄격한 선별: 이질적 컬럼, d/n < 1/10, 실제 데이터)
- **트리**: RF, GBT/HistGBT, XGBoost
- **딥러닝**: MLP, ResNet, FT-Transformer, SAINT
- 모든 모델에 **약 400회 random search**, bootstrap 분산 추정

### 핵심 결과

**성능 순위 (일관적):**
1. GradientBoostingTree / XGBoost
2. RandomForest
3. FT-Transformer (딥러닝 중 최선)
4. SAINT
5. ResNet / MLP (최하위)

**트리 모델이 우세한 3가지 근본 원인:**

**(1) 뉴럴넷은 과도하게 매끄러운(smooth) 함수를 학습**
- 실제 타겟 함수는 불규칙(irregular). 트리는 piece-wise constant로 불규칙 패턴에 강함.
- 타겟을 인위적으로 smooth하게 만들면 뉴럴넷 성능 유지, 트리 성능 하락 → 확인

**(2) 뉴럴넷(MLP)은 비정보적 피처에 취약**
- 비정보적 피처 제거 시 MLP-트리 격차 줄어듦, 추가 시 격차 벌어짐
- FT-Transformer와 트리는 상대적으로 강건

**(3) 데이터는 회전 불변이 아님**
- MLP는 회전 불변 → 테이블 데이터에서 불리 (각 피처가 고유 의미)
- FT-Transformer의 Feature Tokenizer가 회전 불변성을 깨뜨려 MLP보다 우수

### 우리 프로젝트 적용 시사점
- **우리 데이터가 트리 모델 우세 조건에 정확히 부합**:
  - 비정보적 피처 다수 (최대 target 상관 0.037)
  - 불규칙 타겟 (zero-inflated)
  - 이질적 피처 (연속형+이산형 혼합)
- **피처 선택이 딥러닝에 특히 중요**: Boruta/Null Importance로 줄인 후에야 딥러닝 시도 가치
- **FT-Transformer가 딥러닝 중 최선**: 독립 사용보다 앙상블 구성원으로 활용

## 4-3. A Survey on Deep Tabular Learning (2024)

### 논문 정보
- **저자**: Shriyank Somvanshi 외 (Texas State University)
- **학회**: arXiv:2410.12034

### 연구 배경 및 목적
2016~2024년 테이블 데이터 딥러닝 모델 30개 이상의 역사적 진화 과정을 체계적으로 분석하는 포괄적 서베이.

### 주요 모델별 특성

| 모델 | 핵심 기법 | 장점 | 한계 |
|------|----------|------|------|
| **TabNet** (2021) | 순차적 attention 피처 선택 | 해석 가능, 자기지도학습 | XGBoost보다 튜닝 어렵고 학습 느림 |
| **NODE** (2020) | 미분 가능한 oblivious DT 앙상블 | GBDT를 신경망으로 일반화 | 많은 HPO 필요 |
| **FT-Transformer** (2021) | 피처 토큰화 + Self-Attention | 복잡한 피처 상호작용 포착 | attention quadratic scaling |
| **SAINT** (2021) | self + intersample attention | 결측치/노이즈에 강건 | 계산 비용 높음 |
| **TabR** (2023) | retrieval-augmented KNN | 예측 정확도 향상 | - |
| **MambaTab** (2024) | structured state-space model | 장거리 의존성, 진화하는 피처 | - |

### 테이블 데이터에 딥러닝이 어려운 3가지 근본 원인
1. **소규모 샘플**: 과적합 위험
2. **고차원성**: 희소/무관 피처 다수
3. **비공간적 관계**: CNN/RNN의 inductive bias 불작동

### 서베이 결론
- **중간 규모 데이터셋에서 트리 기반 모델이 여전히 SOTA**
- 딥러닝이 우세한 조건: 매우 큰 데이터셋, 복잡한 피처 상호작용, 다중모달, transfer learning
- 정규화 cocktail(Mixup, Dropout, Weight Decay)을 적절히 적용하면 MLP도 GBDT와 경쟁 가능

### 우리 프로젝트 적용 시사점

**데이터 특성 매칭:**

| 특성 | 트리 유리? | DL 유리? |
|------|:---------:|:--------:|
| ~44K samples | O | - |
| ~6,700 features | O | FT-Transformer 일부 |
| 피처-타겟 상관 극히 낮음 | O | X (MLP 취약) |
| Zero-inflated target | O (불규칙 함수) | X (smooth 편향) |
| 수치형+이산형 혼합 | O (네이티브) | 인코딩 필요 |

**권장 모델 우선순위:**
1. **LightGBM, XGBoost** (필수)
2. **FT-Transformer** (피처 선별 후 ~300~500개, 앙상블용)
3. **TabNet** (앙상블 다양성 확보용)
4. **Stacking/Weighted Ensemble** (LightGBM + XGBoost + FT-Transformer)

## 4-4. A Comprehensive Benchmark of Machine and Deep Learning Across Diverse Tabular Datasets (2024)

- **저자**: Assaf Shmuel, Oren Glickman, Teddy Lazebnik
- **학회**: arXiv:2408.14817
- **URL**: https://arxiv.org/abs/2408.14817

### 핵심 결과

| 모델 | 최고 성능 횟수 (111개 중) |

→ **"랜덤 테이블 데이터에서는 ML(트리) 모델이 안전한 선택"**

| 조건 | DL 유리 |
|------|---------|

| 높은 첨도(kurtosis) | **강하게** O |
| 분류 문제 | O (회귀보다) |
| Long-tailed 분포 | O |

**메타학습 모델**: 20개 메타피처(행수, 열수, 첨도, 엔트로피, 왜도, PCA 성분 수, 평균 Pearson 상관 등)로 ML vs DL 우위를 예측. H2O-DL 모델이 86.1% 정확도, AUC 0.78 달성.

### 우리 프로젝트 적용 시사점

**우리 데이터 메타피처 분석:**

| 메타피처 | 우리 데이터 | DL 유리? |
|---------|-----------|---------|
| 행 수 | ~26K (train unit) | 중간 → 약간 ML 유리 |
| 열 수 | ~6,700 (집계 후) | 많음 → **DL 유리** |
| 첨도 | 높음 (왜도 |skew|>2: 314개) | **DL 유리** |
| 과제 유형 | 회귀 | ML 유리 |
| 타겟 분포 | zero-inflated, long-tailed | **DL 유리** |
| 피처-타겟 상관 | 극히 낮음 (max 0.037) | 불확실 |

→ **혼합 신호**: 열 수와 첨도는 DL 유리이지만, 회귀 과제와 중간 샘플 수는 ML 유리. **양쪽 모두 시도하되 트리를 기본으로, DL을 앙상블 보조로 사용**하는 전략이 적합.

- **TabNet은 111개 중 0회 최고**: 단독 사용 부적합 확인. 앙상블 구성원으로만 가치
- **CatBoost 고려**: 19/111로 XGBoost(10/111)보다 자주 최고. 우리 baseline에 CatBoost 포함 타당
- **AutoGluon 활용**: 자동 앙상블 프레임워크가 35%에서 최고 → 빠른 baseline 구축에 활용 가능

# 5. 피처 선택 / 이상치 처리 (4편)

## 5-1. Feature Selection with the Boruta Package (2010)

### 논문 정보
- **저자**: Miron B. Kursa, Witold R. Rudnicki (University of Warsaw)
- **학회**: Journal of Statistical Software, Vol. 36, Issue 11

### 연구 배경 및 목적
기존 피처 선택은 **minimal-optimal**(최소 피처로 최고 성능)에 집중했으나, 이 논문은 **all-relevant**(유의미한 모든 피처 식별) 문제를 다룬다. 현상의 메커니즘을 이해하려면 관련된 모든 피처를 알아야 하기 때문.

### 데이터
- **Ozone**: 12개 속성 (UCI)
- **Madelon**: 500개 속성 (NIPS 2003 Feature Selection Challenge). 실제 중요 20개 + 480개 랜덤 노이즈
- 스케일링 실험: 250~2,000 객체, 500~3,500 속성 (64개 테스트셋)

### 방법론 (Boruta 알고리즘)

1. 모든 변수의 셔플된 복사본(**Shadow attribute**) 생성
2. 확장된 데이터에서 Random Forest 실행, Z-score 계산
3. Shadow 속성 중 **최대 Z-score(MZSA)** 확인
4. MZSA보다 유의하게 **높은** 속성 → "important" 확정
5. MZSA보다 유의하게 **낮은** 속성 → "unimportant" 제거
6. 모든 속성에 판정이 날 때까지 반복

**시간 복잡도**: O(P × N), 속성 수에 대해 선형. ~100만 (속성×객체)당 약 1시간.

### 핵심 결과

**Madelon**: 500개 중 정확히 **20개** important 확정, 480개 rejected (96% 감소)
- 전체 속성 사용 시 테스트 오류 ~0.28 → Boruta 선택 속성 시 **~0.10**
- 단일 RF importance보다 **더 안정적** (비중요 속성이 중요 속성보다 높은 점수 받는 경우 없음)

### 우리 프로젝트 적용 시사점
- **All-relevant 접근법의 가치**: 단일 피처-타겟 상관이 max 0.037으로 극히 약하므로, 피처 **조합**에서 나오는 약한 신호를 포착하는 Boruta가 필터 방법보다 유용
- **~6,700개에서 적용 가능**: 시간 복잡도가 선형. 다만 사전 필터링(분산/상관) 후 적용이 효율적
- Python `BorutaPy`: `RandomForestRegressor` 지원, `max_iter=200`, `n_estimators='auto'`로 설정

## 5-3. Improving Electronic Sensor Reliability by Robust Outlier Screening (2013)

### 논문 정보
- **저자**: Manuel J. Moreno-Lizaranzu (Freescale Semiconductor), Federico Cuesta (U of Seville)
- **학회**: Sensors, Vol. 13, pp. 13521-13542

### 연구 배경 및 목적
자동차/의료 전자 센서의 잠재 결함을 웨이퍼 테스트 단계에서 스크리닝. **공간 알고리즘**(GDBC SB, BBBC)과 **Robust DPAT**를 Freescale 생산 환경에서 수년간 실운영.

### 데이터
- **289,080개 다이** (205,671 양품, 83,409 불량, **26개 CQI**)
- **~1,500개 테스트** 항목
- 천만 개 이상의 웨이퍼 처리 실적

### 방법론

**공간 알고리즘:**
- **GDBC** (Good Die in Bad Cluster): 이웃 중 불량 비율이 임계값 초과 시 양품도 불량 처리
- **GDBC SB** (Specific Bins): 특정 결함 유형(bin)만 고려하여 정밀도 향상
- **BBBC** (Bad Bin in Bad Cluster): 특정 bin 클러스터 주변 양품을 불량 처리

**PAT 알고리즘:**
- **DPAT**: mean ± k·σ (이상치에 편향 가능)
- **AEC DPAT**: median + k·(p99-median)·0.43 (강건)
- **NNR** (Nearest Neighbor Residual): 가우시안 가중 이웃 평균과의 잔차에 DPAT 적용. **공간 트렌드 제거**
- **R-DPAT**: Anderson-Darling 정규성 검정 → Grubbs 이상치 제거 → 비정규 시 Johnson 변환 → AEC DPAT 폴백

### 핵심 결과

**34.6% CQI 검출 시 양품 폐기율:**

| 알고리즘 | 양품 폐기율 |
|---------|-----------|
| 무작위 | 48% |
| DPAT (전체 테스트) | 40% (**무작위보다 나쁨!**) |
| NNR (특정 테스트) | 15.1% |
| **Multi-Site DPAT (특정 테스트)** | **8.9%** |

- **전체 1,500개 테스트로 DPAT 적용 시 무작위보다 나쁨** → 피처 선택 필수
- Multi-Site 분리가 가장 큰 성능 향상 (사이트 간 분산 제거)

### 우리 프로젝트 적용 시사점
- **결함 클러스터링과 공간 피처**: 실증적으로 확인 → die_x, die_y, radial_dist의 중요성 뒷받침
- **강건 이상치 처리**: 단순 mean±kσ보다 **중앙값/백분위수 기반(AEC DPAT)** 또는 **Grubbs+Johnson 변환(R-DPAT)**이 우수
- **관련 없는 피처 포함 시 성능 악화**: 1,500개 전체 → 무작위보다 나쁨. **~6,700개에서 철저한 피처 선택 필수**
- **로트별 정규화**: Multi-Site 효과처럼, run_id별 WT 피처 분포 차이를 고려한 **로트별 정규화 또는 로트 집계 통계 피처** 유효
- **NNR의 공간 잔차 아이디어**: die의 WT 값에서 인접 die 가중 평균을 뺀 **잔차(residual)**를 별도 피처로 추가 고려

## 5-4. Redefining Outliers for On-Wafer Electrical Testing (2024)

### 논문 정보
- **저자**: Michihiro Shintani (Kyoto Institute of Technology), Takashi Sato (Kyoto University)
- **학회**: MLCAD 2024 (ACM/IEEE)

### 연구 배경 및 목적
DPAT의 한계(공간 트렌드 미고려)를 **가우시안 프로세스 회귀(GPR)**로 극복. DPAT로 검출 불가능한 3가지 이상치 유형:
- **Chip A**: DPAT 한계 초과 → 검출 가능
- **Chip B**: DPAT 내이지만 주 분포에서 약간 벗어남 → 검출 불가
- **Chip C**: DPAT 내, 분포 중간이지만 **인접 칩과 크게 다름** → 절대 검출 불가

### 데이터
- Power device: SiC 웨이퍼 1장, 435개 칩
- LSI: CMOS LSI, 25개 웨이퍼, 각 ~4,000개 칩
- EGPR 실험: 40nm 자동차용 LSI, 60개 웨이퍼, 각 ~4,000개 다이. 전문가 3단계 라벨링 (alpha/beta/gamma/양품)

### 방법론

**GPR 기반 이상치 검출:**
- 칩 좌표를 입력, 측정값을 출력으로 GPR 학습 → 칩별 공간 트렌드 고려한 **적응적 임계값**
- RBF 커널: k = θ₁·exp(-(xi-xj)²/θ₂)
- **DPAT과의 차이**: DPAT는 모든 칩에 동일 임계값, GPR은 칩별 다른 임계값

**EGPR (Ensemble GPR):**
1. 데이터를 S개 배치로 분할 (Poisson Disk Sampling으로 이상치 분산)
2. 배치별 GPR 학습 (해당 배치 제외 나머지로 예측)
3. S개 예측의 **중앙값(median)**으로 앙상블 → 이상치 편향 제거
4. 측정값이 μ̃ ± Z·σ̃ 범위 밖이면 이상치

### 핵심 결과

| 방법 | Rank alpha (148) | Rank beta (58) | Rank gamma (95) |
|------|:-:|:-:|:-:|
| GPR (S=1) | 100% | 87.9% | 62.1% |
| **EGPR (S=5)** | **100%** | **96.6%** | **73.7%** |

- EGPR이 잠재 이상치(beta, gamma) 검출에서 GPR 대비 **최대 15.6% 향상**
- 계산 시간: GPR 54.1초 → EGPR(S=7) **0.76초** (98.6% 감소, GPU)

### 우리 프로젝트 적용 시사점
- **공간 트렌드의 중요성**: 동심원 패턴 등 → radial_dist, die_x, die_y 피처의 이론적 근거
- **GPR 잔차를 피처로 활용**: WT 데이터에서 각 die의 측정값과 GPR 공간 트렌드의 **잔차(residual)**를 새 피처로 추가 가능. 공간 트렌드에서 크게 벗어난 die = 불량 위험 높음
- **칩별 적응적 임계값**: 전역 IQR보다 **run_id/웨이퍼/위치별 국소 기준**이 더 효과적
- **Rank gamma (분포 내이지만 공간 이상)**: 전체 분포 기준 정상이지만 인접 die와 다른 die → die-level 공간 잔차 피처가 이를 포착

# 6. 핵심 ML 모델 원논문 (4편)

## 6-1. XGBoost: A Scalable Tree Boosting System (2016)

> *웹 공개 정보 기반 요약*

### 논문 정보
- **저자**: Tianqi Chen, Carlos Guestrin (University of Washington)
- **학회**: KDD'16 (ACM SIGKDD), pp. 785-794
- **URL**: https://dl.acm.org/doi/10.1145/2939672.2939785 / arXiv:1603.02754

### 핵심 기술

**1. 정규화된 목적 함수 (Regularized Objective)**
```
Obj = Σ l(yi, ŷi) + Σ Ω(fk)
Ω(f) = γT + ½λ||w||²
```
- `l`: 손실 함수 (MSE, logloss 등)
- `Ω`: 정규화 항 (트리 복잡도 패널티)
- `γ`: 리프 노드 수(T)에 대한 패널티 → 과적합 방지
- `λ`: 리프 가중치(w)의 L2 정규화 → 과적합 방지

**2. Sparsity-Aware Algorithm**
- 결측값과 희소 데이터를 효율적으로 처리
- 결측값이 있는 샘플을 자동으로 최적 방향으로 분기 (default direction learning)
- 별도 imputation 없이 결측값 처리 가능

**3. Weighted Quantile Sketch**
- 근사 분할(approximate split finding)을 위한 가중 분위수 알고리즘
- 대규모 데이터에서 정확한 분할점을 효율적으로 찾음
- 분산 환경에서도 merge 가능한 알고리즘

**4. 시스템 최적화**
- **Column Block**: 데이터를 열(column) 기준으로 정렬하여 캐시 효율 극대화
- **Cache-Aware Access**: 메모리 접근 패턴 최적화
- **Out-of-Core Computation**: 메인 메모리 초과 데이터도 디스크 I/O로 처리
- **Data Compression + Sharding**: 분산 학습 지원

**5. 추가 기법**
- **Shrinkage (학습률)**: 각 트리의 기여도를 η로 축소 → 과적합 방지
- **Column Subsampling**: Random Forest의 피처 서브샘플링을 부스팅에 도입 → 과적합 방지 + 속도 향상

### 우리 프로젝트 적용 시사점
- **결측값 자동 처리**: 우리 데이터의 1,087개 피처 모두에 결측 존재. XGBoost는 별도 imputation 없이도 학습 가능 (성능 비교 실험 가치)
- **정규화 파라미터 γ, λ**: ~6,700개 고차원 피처에서 과적합 방지에 핵심. Optuna HPO로 최적화 필수
- **Column Subsampling**: 고차원 데이터에서 colsample_bytree를 0.3~0.7로 설정하면 속도+성능 동시 향상
- Kaggle/산업 경진대회에서 가장 많이 사용되는 모델 → 우리 경진대회에서도 핵심 baseline

## 6-2. LightGBM: A Highly Efficient Gradient Boosting Decision Tree (2017)

> *웹 공개 정보 기반 요약*

### 논문 정보
- **저자**: Guolin Ke, Qi Meng, Thomas Finley, Taifeng Wang, Wei Chen, Weidong Ma, Qiwei Ye, Tie-Yan Liu (Microsoft Research)
- **학회**: NeurIPS 2017 (NIPS), pp. 3146-3154
- **URL**: https://proceedings.neurips.cc/paper/2017/hash/6449f44a102fde848669bdd9eb6b76fa-Abstract.html

### 핵심 기술

**1. GOSS (Gradient-based One-Side Sampling)**
- 그래디언트가 큰 샘플(잘 맞추지 못하는 샘플)은 모두 유지
- 그래디언트가 작은 샘플(이미 잘 맞추는 샘플)은 랜덤 서브샘플링
- 정보 손실을 최소화하면서 학습 데이터 크기를 대폭 축소
- **효과**: 학습 속도 향상, 정확도 유지

**2. EFB (Exclusive Feature Bundling)**
- 상호 배타적인 피처들(동시에 non-zero가 아닌 피처)을 하나로 번들링
- 고차원 희소 데이터에서 실질적 피처 수를 대폭 축소
- 그래프 coloring 알고리즘으로 번들 결정
- **효과**: 피처 수 감소 → 분할점 탐색 비용 절감

**3. Leaf-wise (Best-first) Tree Growth**
- 기존 GBDT의 level-wise(층별) 성장 대신, **가장 큰 loss 감소를 주는 리프를 우선 분할**
- 같은 리프 수에서 더 낮은 loss 달성
- 과적합 위험이 있으므로 max_depth로 제어

**4. Histogram-based Algorithm**
- 연속 피처를 이산 bin으로 히스토그램화
- 분할점 탐색 시 bin 단위로 계산 → O(#data × #features) → O(#bins × #features)
- 메모리 효율적

### 핵심 결과
- 기존 GBDT 대비 **20배 이상 학습 속도 향상**, 거의 동일한 정확도
- 대규모 데이터셋에서 XGBoost보다 빠르고 메모리 효율적

### 우리 프로젝트 적용 시사점
- **~44K × ~6,700 데이터에 최적**: GOSS + EFB가 대규모 피처에서 속도와 정확도를 동시에 확보
- **EFB가 우리 데이터에 특히 유용**: 1,087개 WT 피처 중 상당수가 희소하거나 상호 배타적일 수 있음 → LightGBM이 자동으로 번들링
- **Leaf-wise 성장**: 비정보적 피처가 많은 우리 데이터에서, loss 감소가 큰 분할을 우선 수행하므로 효율적
- **Optuna HPO 핵심 파라미터**: `num_leaves`, `max_depth`, `learning_rate`, `min_child_samples`, `colsample_bytree`, `reg_alpha`, `reg_lambda`
- **Category 피처 네이티브 지원**: 이산형 피처(고유값 ≤20인 391개)를 원-핫 인코딩 없이 직접 처리 가능

## 6-3. TabNet: Attentive Interpretable Tabular Learning (2021)

> *웹 공개 정보 기반 요약*

### 논문 정보
- **저자**: Sercan O. Arik, Tomas Pfister (Google Cloud AI)
- **학회**: AAAI Conference on Artificial Intelligence, Vol. 35(8), pp. 6679-6687
- **URL**: https://ojs.aaai.org/index.php/AAAI/article/view/16826 / arXiv:1908.07442

### 핵심 기술

**1. Sequential Attention (순차적 주의 메커니즘)**
- 각 decision step에서 **어떤 피처에 집중할지 동적으로 선택**
- Sparsemax 활성화 함수로 소수의 피처만 선택 (sparse feature selection)
- 각 step의 피처 선택 결과가 다음 step에 반영 (sequential dependency)

**2. Feature Reusage (피처 재사용)**
- 이전 step에서 선택된 피처 정보를 다음 step에 전달
- Relaxation factor로 동일 피처의 반복 선택 정도를 조절

**3. Attentive Transformer**
```
Step 1: 피처 서브셋 A1 선택 → 처리 → 부분 출력
Step 2: 피처 서브셋 A2 선택 → 처리 → 부분 출력
...
Step N: 피처 서브셋 AN 선택 → 처리 → 부분 출력
최종 출력 = Σ 부분 출력
```

**4. Self-Supervised Learning**
- Encoder-Decoder 구조로 마스킹된 피처를 복원하는 사전학습
- 라벨 없는 데이터를 활용하여 피처 표현 학습
- **최초로 테이블 데이터에서 자기지도학습의 효과를 보임**

**5. Interpretability (해석 가능성)**
- 각 decision step의 attention mask → 어떤 피처가 예측에 기여했는지 시각화
- Global feature importance = 모든 샘플의 attention mask 평균

### 핵심 결과
- Forest Cover Type, Poker Hand, Sarcos, Higgs 등에서 기존 DNN 및 일부 트리 모델 대비 우수
- 자기지도학습으로 라벨 부족 시 성능 향상
- 단, 후속 연구(Shwartz-Ziv 2021, Grinsztajn 2022)에서 XGBoost 대비 일관된 우위를 보이지 못함

### 우리 프로젝트 적용 시사점
- **피처 선택 내장**: ~6,700개 피처에서 자동으로 중요 피처를 선택. 별도 피처 선택 없이도 작동
- **해석 가능성**: attention mask로 어떤 WT 피처가 RCC 예측에 기여하는지 파악 → 공정 개선 인사이트
- **자기지도학습**: validation/test split의 X 데이터(Y 없음)를 사전학습에 활용 가능. 우리 데이터는 train 60% + val+test 40%(Y 비공개)이므로 이점이 있을 수 있음
- **단독 사용보다 앙상블 구성원으로 활용**: 4-1의 결론에 따라, XGBoost와 앙상블할 때 가장 효과적

## 6-4. Revisiting Deep Learning Models for Tabular Data / FT-Transformer (2021)

> *웹 공개 정보 기반 요약*

### 논문 정보
- **저자**: Yury Gorishniy, Ivan Rubachev, Valentin Khrulkov, Artem Babenko (Yandex Research)
- **학회**: NeurIPS 2021
- **URL**: https://proceedings.neurips.cc/paper/2021/hash/9d86d83f925f2149e9edb0ac3b49229c-Abstract.html / arXiv:2106.11959
- **코드**: https://github.com/yandex-research/rtdl-revisiting-models

### 핵심 기술

**1. Feature Tokenizer (피처 토큰화)**
- 각 수치형/범주형 피처를 개별 토큰(임베딩 벡터)으로 변환
- 수치형: `ti = bi + xi · wi` (선형 임베딩)
- 범주형: lookup table 임베딩
- **핵심**: 피처별 독립 임베딩 → 피처의 원래 의미(방향성)를 보존 → 회전 불변성을 깨뜨림

**2. Transformer Backbone**
- Feature Tokenizer의 출력 토큰들에 **Multi-Head Self-Attention** 적용
- 피처 간 복잡한 상호작용(interaction)을 attention으로 포착
- [CLS] 토큰을 추가하여 최종 예측에 사용

**3. ResNet Baseline**
- 논문에서 ResNet-like 아키텍처(MLP + skip connections)가 **강력한 기본 baseline**임을 발견
- 기존 연구에서 과소평가되었던 단순 아키텍처의 재평가

### 핵심 결과
- FT-Transformer가 **대부분의 테이블 데이터 태스크에서 다른 딥러닝 모델을 능가**
- 그러나 **GBDT와 비교하면 "보편적으로 우월한 해법은 없다"** — 데이터에 따라 다름
- Grinsztajn et al.(2022)에서 **딥러닝 중 유일하게 트리와 경쟁 가능**한 모델로 인정

### 우리 프로젝트 적용 시사점
- **Feature Tokenizer가 핵심**: 각 피처를 독립 토큰으로 변환하여 피처별 의미를 보존. 이것이 MLP보다 테이블 데이터에 적합한 이유 (4-2의 원인 분석 3번)
- **Attention의 quadratic scaling 문제**: 피처 수 K에 대해 O(K²) 복잡도 → ~6,700개 피처에서는 비현실적. **Boruta 등으로 ~300~500개로 줄인 후 적용** 필수
- **앙상블에서 다양성 확보**: LightGBM/XGBoost와 FT-Transformer는 다른 inductive bias를 가지므로, 앙상블 시 다양성 확보에 효과적
- **코드 공개**: GitHub에서 즉시 사용 가능

## 2-5. Tweedie Gradient Boosting for Extremely Unbalanced Zero-inflated Data (2019)

### 논문 정보
- **저자**: He Zhou (University of Minnesota), Wei Qian (University of Delaware), Yi Yang (McGill University)
- **학회**: arXiv:1811.10192, Communications in Statistics - Simulation and Computation, Vol. 51, No. 9

### 연구 배경 및 목적
보험 청구 데이터처럼 **zero 비율이 90% 이상**인 극단적 zero-inflated 데이터에서 전통 Tweedie GLM조차 불충분한 문제를 해결. Zero-inflated Tweedie 모델에 gradient tree-boosting을 결합한 **EMTboost** 방법론 제안.

### 방법론
- **Zero-Inflated Tweedie 모델**: 확률 q로 Tweedie 분포, 확률 (1-q)로 정확히 0인 mixture model
- **EMTboost 알고리즘**: EM(Expectation-Maximization) + blockwise coordinate descent + gradient tree boosting을 통합
  - E-step: 각 관측치가 structural zero인지 Tweedie에서 나온 zero인지의 posterior 확률 계산
  - M-step: (1) 분산/power 파라미터 업데이트 (coordinate descent), (2) Tweedie 평균 함수 업데이트 (gradient tree boosting)
- 정규화: ridge/soft-thresholding으로 zero mass 확률 q 추정 안정화
- 비선형성, 불연속성, 고차 상호작용을 자동 포착 (트리 기반)

### 핵심 결과

**시뮬레이션 (Case 1.1, zero 비율 ~65%)**:

| 모델 | Gini_a | MAD |
|------|--------|-----|
| Grabit (Tobit boosting) | 0.345 | 0.316 |
| TDboost (Tweedie boosting) | 0.459 | 0.294 |
| **EMTboost** | **0.516** | **0.248** |

- EMTboost가 모든 시뮬레이션 설정(two-interaction, Friedman 함수)에서 Gini 및 MAD 최고
- zero mass 확률 q와 power 파라미터 ρ를 동시 추정 가능 (TDboost/Grabit는 불가)

**실데이터 (AutoClaim 보험 청구, zero 비율 94.6%)**:
- EMTboost가 TDboost, Grabit 대비 Gini_a, Gini_b 모두 최고

### 우리 프로젝트 적용 시사점
- **Y의 zero 비율 70.8%**: EMTboost의 ZI-Tweedie가 직접 적용 가능한 구조
- **Stage 7-B의 이론적 근거**: Two-Stage 대신 단일 모델로 ZI 처리하는 대안 접근법 제공
- **LightGBM/XGBoost의 `objective='tweedie'`**: tweedie_variance_power 1.0~1.5 실험 근거. EMTboost는 power를 자동 추정
- **Structural zero vs. sampling zero 분리**: EM의 posterior 확률로 "진짜 양품(structural zero)"과 "우연히 불량 안 난 제품"을 구분

## 2-6. Enhanced Gradient Boosting for Zero-Inflated Insurance Claims and Comparative Analysis of CatBoost, XGBoost, and LightGBM (2024)

### 논문 정보
- **저자**: Banghee So (Towson University)
- **학회**: arXiv:2307.07771, Scandinavian Actuarial Journal, Vol. 2024(10), pp. 1013-1035

### 연구 배경 및 목적
보험 청구 빈도 모델링에서 excess zero 문제를 해결하기 위해 **Zero-Inflated Poisson(ZIP) boosted tree** 모델을 제안하고, CatBoost/XGBoost/LightGBM 3개 라이브러리를 체계적으로 비교.

### 방법론
- **ZIP Boosted Tree 모델**: inflation probability p와 Poisson mean μ를 동시에 gradient boosting으로 학습
  - 변형 1: p와 μ가 독립 (ZIP-I)
  - 변형 2: p가 μ의 함수 (ZIP-D) — 더 간결하고 해석 용이
- **3개 GBT 라이브러리 비교**: CatBoost, XGBoost, LightGBM
  - CatBoost: Ordered Boosting으로 target leakage 방지, 범주형 피처 네이티브 처리
  - LightGBM: GOSS + EFB로 속도 최적화
  - XGBoost: Sparsity-Aware Algorithm
- 2개 데이터셋: (1) 프랑스 자동차보험 (freMTPL2freq), (2) 캐나다 텔레매틱스 보험

### 핵심 결과

**freMTPL2freq 데이터셋 (67.8만 policies, zero 비율 ~96%)**:

| 모델 | Poisson Deviance | Zero-Inflated |
|------|-----------------|---------------|
| XGBoost Poisson | 0.3163 | - |
| LightGBM Poisson | 0.3162 | - |
| **CatBoost Poisson** | **0.3155** | - |
| **CatBoost ZIP-D** | - | **최고 적합도** |

**텔레매틱스 데이터셋 (zero 비율 ~97%)**:
- CatBoost이 모든 설정에서 최고 성능
- ZIP 모델이 standard Poisson보다 BIC 기준 우수
- CatBoost의 SHAP 기반 해석이 가장 직관적

### 우리 프로젝트 적용 시사점
- **CatBoost의 ZI 데이터 우수성**: Y=0 70.8%인 우리 데이터에 CatBoost을 Stage 7-B의 핵심 모델로 배치하는 근거
- **ZIP-D 구조 참고**: inflation probability를 mean의 함수로 모델링하면 파라미터가 적어 과적합 위험 감소
- **SHAP 해석**: CatBoost + SHAP 조합으로 "어떤 WT 피처가 불량에 기여하는지" 해석 → Phase 4 대시보드에 활용

## 2-7. Zero-Inflated Tweedie Boosted Trees with CatBoost for Insurance Loss Analytics (2024)

### 논문 정보
- **저자**: Banghee So (Towson University), Emiliano A. Valdez (University of Connecticut)
- **학회**: arXiv:2406.16206, Applied Soft Computing (2025). **2024 ASTIN Best Paper Award** 수상
- **URL**: https://arxiv.org/abs/2406.16206

### 연구 배경 및 목적
보험 aggregate claim 모델에서 Tweedie 회귀 모델의 한계(zero claim 과소 적합)를 해결. **Zero-Inflated Tweedie + CatBoost** 프레임워크로 zero claim과 positive claim을 통합 모델링.

### 방법론
- **ZI-Tweedie 모델**: zero-inflation 확률 π₀을 Tweedie mean μ의 함수로 설정
  - `π₀(μ) = exp(-λ)` where `λ = μ^(2-ρ) / (φ(2-ρ))` (Tweedie의 Poisson 도착 강도)
  - ZI 부분과 Tweedie 부분이 하나의 파라미터 μ로 연결 → 단일 목적함수로 최적화 가능
- **CatBoost 프레임워크**: custom Tweedie loss를 CatBoost에 구현하여 gradient boosting
- **Compositional feature** 처리: 텔레매틱스의 비율 데이터를 isometric log-ratio(ILR) 변환으로 처리

### 핵심 결과

| 모델 | Log-Likelihood | AIC |
|------|---------------|-----|
| Tweedie GLM | -84,419 | 168,855 |
| Tweedie CatBoost | -83,771 | 167,565 |
| **ZI-Tweedie CatBoost** | **-83,605** | **167,238** |

- ZI-Tweedie CatBoost이 모든 적합도 지표(LL, AIC, BIC)에서 최고
- Zero claim 예측 정확도가 standard Tweedie 대비 유의미하게 향상
- CatBoost의 SHAP 분석으로 피처별 기여도 해석 가능

### 우리 프로젝트 적용 시사점
- **단일 모델 ZI 접근법의 최신 SOTA**: Two-Stage(분류+회귀) 대신 ZI-Tweedie 단일 모델도 강력한 대안
- **CatBoost custom loss 구현 참고**: `objective='tweedie'`에 ZI 확장을 custom으로 구현하는 패턴
- **Stage 7-B 비교 실험 설계**: Two-Stage(분류+회귀) vs. ZI-Tweedie 단일 모델을 RMSE로 비교하여 더 나은 쪽 채택

## 2-8. Dealing with Zero-Inflated Data: Achieving SOTA with a Two-Fold ML Approach (2025)

### 논문 정보
- **저자**: Jože M. Rožanec 외 (Jožef Stefan Institute, Slovenia)
- **학회**: arXiv:2310.08088, Engineering Applications of Artificial Intelligence (Elsevier, 2025)

### 연구 배경 및 목적
Zero-inflated 데이터에서 **Two-fold(2단계) ML 모델**이 기존 SOTA 단일 모델을 대폭 능가함을 실증. 가전제품 분류 및 공항 셔틀 수요 예측 두 가지 실제 use case에서 검증.

### 방법론
- **Two-fold 구조**:
  - Fold 1 (분류): zero vs. non-zero 이진 분류 (Histogram-based Gradient Boosting)
  - Fold 2 (회귀/분류): non-zero 데이터만으로 학습 (SVR, MLP, LR 등)
  - 최종 예측: 분류기가 "non-zero"로 판단한 경우에만 회귀 예측 출력, 그 외 0
- **최적 threshold**: Youden index (`J = sensitivity + specificity - 1`)로 분류 임계값 최적화
- **Target 변환**: 회귀 target에 `log(1+x)` 변환 적용하여 왜도 감소

### 핵심 결과

**공항 셔틀 수요 예측 (zero 비율 ~60%)**:

| 모델 | MASE (낮을수록 좋음) |
|------|-----|
| HGB Regressor (단일) | baseline |
| **HGB Classifier + SVR (Two-fold)** | **통계적으로 유의하게 최저** |

- Two-fold 모델이 **모든 케이스**에서 단일 회귀 모델을 능가 (통계적 유의성 확인)
- **가전제품 분류**: Precision +27%, Recall +34%, F1 +49%, AUC ROC +27% 향상
- Two-fold 모델이 에너지 효율도 4배 우수 (적은 데이터로 학습)

### 우리 프로젝트 적용 시사점
- **Two-Stage 전략의 최신 실증 근거**: Stage 7-B(분류→회귀) 전략이 2025년 SOTA임을 확인
- **Youden index**: Stage 1 분류기의 threshold를 0.5 대신 Youden index로 최적화하면 RMSE 추가 개선 가능
- **log(1+x) target 변환**: Y>0 서브셋의 회귀에서 target 변환 실험 근거
- **HGB + SVR 조합**: Stage 1에 GBT 분류기, Stage 2에 SVR도 비교 대상에 포함

## 2-9. ZIPBoost/ZILBoost: A Systematic Approach for Learning Imbalanced Data Through Zero-Inflated Boosting (2024)

### 논문 정보
- **저자**: Yeasung Jeong, Kangbok Lee, Young Woong Park, Sumin Han
- **학회**: Machine Learning (Springer), Vol. 113, pp. 8233-8299 (2024)

### 연구 배경 및 목적
Imbalanced 데이터를 **two-regime process**로 체계적으로 모델링. Regime 0에서 과잉 zero(majority)를, Regime 1에서 minority class를 생성하는 구조. 이를 boosting과 결합한 **ZIPBoost**(probit)와 **ZILBoost**(logit) 알고리즘 제안.

### 방법론
- **Two-regime 프로세스**: 
  - Regime 0: split probit(logit)로 zero mass 모델링
  - Regime 1: ordinary probit(logit)로 outcome 모델링
  - 각 regime에 다른 예측변수 사용 가능
- **Boosting 전략**: AdaBoost처럼 오분류 가중치 업데이트 + cyclic coordinate descent (Newton-Raphson)
- **모델 선택 기준**: 데이터 분포의 excess kurtosis에 따라 ZIPBoost(platykurtic) vs. ZILBoost(leptokurtic) 선택
- 수렴 증명 및 시간 복잡도 분석 제공

### 핵심 결과

**Monte Carlo 시뮬레이션**:
- ZIPBoost/ZILBoost가 AdaBoost, LogitBoost, SMOTE, EasyEnsemble 대비 높은 Accuracy 및 F1

**Keel 불균형 데이터셋 (9개)**:
- ZIPBoost가 **평균 순위 1위** (9개 데이터셋 기준)

**M&A 데이터 (zero 비율 ~97%)**:
- ZIPBoost: AUC 0.751, ZILBoost: 0.743 → AdaBoost(0.698), SMOTE+LR(0.717) 대비 우수

### 우리 프로젝트 적용 시사점
- **Two-regime 관점 도입**: Y=0(정상 제품)은 Regime 0, Y>0(불량 제품)은 Regime 1로 분리하는 이론적 틀
- **Stage 1 분류기 개선**: 단순 이진 분류 대신 ZIPBoost 구조를 적용하면 regime별로 다른 피처를 활용 가능
- **Kurtosis 기반 모델 선택**: 우리 데이터의 excess kurtosis를 확인하여 probit vs. logit 모델 선택

## 5-5. Permutation Importance: A Corrected Feature Importance Measure (2010)

### 논문 정보
- **저자**: André Altmann, Laura Toloşi, Oliver Sander, Thomas Lengauer (Max Planck Institute for Informatics)
- **학회**: Bioinformatics, Vol. 26, No. 10, pp. 1340-1347

### 연구 배경 및 목적
Random Forest의 Variable Importance(VI)와 Gini Importance(GI)가 **카테고리 수가 많은 변수를 편향적으로 선호**하는 문제를 교정. 반복적 target 순열을 통해 각 피처의 중요도에 대한 **p-value**를 부여하는 **PIMP(Permutation Importance)** 방법 제안.

### 방법론
1. 원본 데이터로 RF 학습 → 각 피처의 importance 측정 (VI 또는 GI)
2. Target 벡터를 S회 순열(셔플) → 각 순열마다 RF 재학습 → S개의 "null importance" 값 획득
3. Null importance 분포에 확률 분포 적합 (정규 분포 또는 비모수)
4. 원본 importance의 p-value 계산 → **유의미한 피처만 선택** (p < 0.05)

### 핵심 결과

**시뮬레이션 (1,000개 피처 중 5개만 유의미)**:
- PIMP로 5개 유의미 피처를 정확히 복원 (p < 0.05)
- VI, GI 단독으로는 카테고리 수 많은 무의미 피처에 높은 점수 부여 → PIMP가 교정

**실데이터 (HIV 약물 내성 예측)**:
- PIMP 기반 피처 선별 후 RF → 기존 모델 대비 **예측 정확도 향상**
- 무의미 피처 제거로 모델 단순화 + 해석력 향상

### 우리 프로젝트 적용 시사점
- **Stage 6 Permutation Importance 검증의 원논문**: `sklearn.inspection.permutation_importance`의 이론적 근거
- **Null Importance 기법의 원리**: target을 셔플한 상태에서의 importance와 비교 → CLAUDE.md의 "Null Importance" 전략이 이 논문에 기반
- **투표 기반 피처 선택에서 한 축**: PIMP p-value가 유의한 피처 = Boruta/LightGBM importance와 교차 검증 가능
- **편향 교정**: 1,087개 피처 중 이산형(고유값 ≤20) 391개가 있으므로, 카테고리 편향 교정이 중요

## 6-5. CatBoost: Unbiased Boosting with Categorical Features (2018)

### 논문 정보
- **저자**: Liudmila Prokhorenkova, Gleb Gusev, Aleksandr Vorobev, Anna Veronika Dorogush, Andrey Gulin (Yandex)
- **학회**: NeurIPS 2018 (32nd Conference on Neural Information Processing Systems)
- **URL**: https://arxiv.org/abs/1706.09516

### 연구 배경 및 목적
기존 모든 gradient boosting 구현체에 존재하는 **prediction shift(예측 편향)** 문제를 규명하고 해결. 두 가지 핵심 알고리즘 기여: **(1) Ordered Boosting** — 순열 기반 학습으로 target leakage 방지, **(2) Ordered Target Statistics** — 범주형 피처의 target encoding에서 누수 방지.

### 방법론

**Prediction Shift 문제 분석:**
- 기존 GBDT는 t번째 트리 학습 시 gradient를 동일 학습 데이터로 계산 → 학습 데이터의 gradient 분포가 테스트 데이터와 다름 (shift)
- Theorem 1로 정량적 증명: 동일 데이터 사용 시 bias ∝ 1/(n-1)

**Ordered Boosting:**
- 데이터에 임의 순열(σ) 적용 → 각 샘플의 gradient는 순열에서 앞선 샘플들로만 학습된 모델로 계산
- s+1개 독립 순열 사용: s개는 트리 구조 결정, 1개는 leaf 값 결정
- Oblivious Decision Trees 사용 (같은 레벨에서 동일 분할 기준) → 과적합 감소, 추론 속도 향상

**Ordered Target Statistics:**
- 범주형 피처를 target 평균으로 인코딩 시 "자기 자신의 target 사용" 문제 → 순열 기반으로 해결
- Greedy TS, Leave-one-out TS, Holdout TS 모두 누수 존재 → Ordered TS가 유일하게 해결

### 핵심 결과

| 데이터셋 | CatBoost Logloss | XGBoost 증가율 | LightGBM 증가율 |
|---------|-----------------|--------------|----------------|
| Adult | 0.270 | +2.2% | +2.4% |
| Amazon | 0.139 | +17% | +17% |
| Click | 0.392 | +1.2% | +1.2% |
| Epsilon | 0.265 | +11% | +1.5% |
| Internet | 0.209 | +7.9% | +6.8% |
| Kick | 0.286 | +3.2% | +3.5% |

- **CatBoost이 8개 데이터셋 모두에서 최고 성능** (logloss 및 zero-one loss)
- Ordered TS가 Greedy/Leave-one-out/Holdout TS 대비 일관 우수 (Amazon에서 Greedy 대비 40% 개선)
- 작은 데이터셋일수록 Ordered Boosting의 이점 증가 (Plain 대비 최대 25% 개선)

### 우리 프로젝트 적용 시사점
- **CatBoost baseline 포함의 원리적 근거**: Ordered Boosting이 target leakage를 이론적으로 해결하므로, 특히 **Target Encoding**(Stage 4.5 방법 B) 사용 시 CatBoost이 자체적으로 누수를 방지
- **run_id, wafer_no 같은 범주형 피처**: CatBoost의 Ordered TS가 자동으로 최적 인코딩 → 수동 Target Encoding보다 안전
- **소규모 데이터에서 이점**: validation(8,749 unit)이 적은 우리 데이터에서 Ordered Boosting의 편향 교정이 더 중요

## 6-6. Optuna: A Next-generation Hyperparameter Optimization Framework (2019)

### 논문 정보
- **저자**: Takuya Akiba, Shotaro Sano, Toshihiko Yanase, Takeru Ohta, Masanori Koyama (Preferred Networks)
- **학회**: KDD 2019 (25th ACM SIGKDD)
- **URL**: https://arxiv.org/abs/1907.10902

### 연구 배경 및 목적
기존 HPO 프레임워크(Hyperopt, Spearmint, SMAC)의 한계를 극복하는 차세대 하이퍼파라미터 최적화 프레임워크. **Define-by-run API**, **효율적 sampling + pruning**, **분산 컴퓨팅 지원**의 3가지 설계 기준 제안.

### 방법론

**1. Define-by-run API:**
- 파라미터 탐색 공간을 정적으로 정의(define-and-run)하는 대신, 목적 함수 실행 중에 동적으로 구성
- 조건부 파라미터(예: 레이어 수에 따라 유닛 수 변경)를 자연스럽게 표현 가능
- Hyperopt 대비 코드 복잡도 50% 이상 감소

**2. Sampling 알고리즘:**
- **TPE (Tree-structured Parzen Estimator)**: 독립 샘플링 + 그룹 분해로 고차원 탐색 효율화
- CMA-ES: 연속 파라미터에 대한 진화 전략
- Random/Grid: 기본 baseline

**3. Pruning 알고리즘:**
- **Successive Halving / Hyperband**: 성능 낮은 trial을 조기 종료하여 리소스 절약
- Median Pruning: 현재 trial이 이전 trials의 중간값보다 나쁘면 중단

**4. 분산 컴퓨팅:**
- 단일 프로세스 ~ 수백 워커까지 확장
- RDB(MySQL, PostgreSQL) 기반 trial 공유

### 핵심 결과
- MNIST에서 Hyperopt 대비 **동일 trial 수에서 더 낮은 error** 달성
- 대규모 실무(Preferred Networks)에서 1년 이상 프로덕션 사용 검증
- MLP 아키텍처 탐색에서 define-by-run이 define-and-run 대비 직관적이고 버그 감소

### 우리 프로젝트 적용 시사점
- **Stage 7-D HPO의 핵심 도구**: CLAUDE.md에서 이미 Optuna + LightGBM 조합을 권장. 이 논문이 그 원리
- **Pruning으로 탐색 시간 단축**: `optuna.integration.LightGBMPruningCallback`으로 비유망 trial 조기 종료
- **Define-by-run의 실용성**: Two-Stage 모델에서 Stage 1/Stage 2 하이퍼파라미터를 한 objective 함수에서 동시 탐색 가능
- **분산 학습**: Colab에서 단일 GPU 실행이지만, 추후 멀티 GPU 환경에서 확장 가능

## 8-1. Delving into Deep Imbalanced Regression (2021)

### 논문 정보
- **저자**: Yuzhe Yang, Kaiwen Zha, Ying-Cong Chen, Hao Wang, Dina Katabi (MIT CSAIL, Rutgers)
- **학회**: ICML 2021 (Proceedings of the 38th International Conference on Machine Learning)
- **URL**: https://arxiv.org/abs/2102.09554

### 연구 배경 및 목적
**Deep Imbalanced Regression (DIR)** 문제를 정의: 연속 타겟에서 특정 값 구간의 샘플이 부족한 불균형 회귀. 분류의 re-sampling/re-weighting을 그대로 적용하면 연속 타겟의 "인접 유사성"을 무시하게 됨. 이를 해결하는 **LDS(Label Distribution Smoothing)**와 **FDS(Feature Distribution Smoothing)** 제안.

### 방법론

**LDS (Label Distribution Smoothing):**
- target 분포를 Gaussian kernel로 smoothing → 인접 target의 밀도를 고려한 re-weighting
- 이산적 bin frequency 대신 연속적 "effective density" 사용
- `weight_i = 1 / (LDS_smoothed_density(y_i))`

**FDS (Feature Distribution Smoothing):**
- feature space에서 인접 target을 가진 샘플들의 feature 통계(mean, covariance)를 Gaussian kernel로 smoothing
- 소수 target 구간의 feature 표현을 인접 다수 구간의 정보로 보정 (calibration)
- 모델의 마지막 feature layer에 적용, end-to-end 학습 가능

**DIR 벤치마크 데이터셋 5개:**
- IMDB-WIKI-DIR (나이 추정), AgeDB-DIR, STS-B-DIR (텍스트 유사도), NYUD2-DIR (깊이), SHHS-DIR (건강)

### 핵심 결과

**IMDB-WIKI-DIR (나이 추정)**:

| 방법 | 전체 MAE | Few-shot MAE | Few-shot GM |
|------|---------|-------------|-------------|
| Vanilla | 8.06 | 26.33 | 20.46 |
| SMOTER | 8.14 | 25.28 | 19.46 |
| SMOGN | 8.03 | 25.93 | 20.12 |
| **SqInv + LDS + FDS** | **7.78** | **22.19** | **12.61** |

- LDS + FDS가 few-shot(데이터 부족) 구간에서 **MAE 4.14 감소, GM 7.85 감소**
- SMOTER/SMOGN은 고차원 이미지에서 오히려 성능 악화
- LDS + FDS는 모든 기존 방법과 조합 가능하며 일관 개선

### 우리 프로젝트 적용 시사점
- **Y>0 서브셋 내에서의 불균형 대응**: Y>0인 29.2% 중에서도 값 분포가 극도로 skewed (대부분 0.001~0.015, max=1.0). DIR 기법으로 이 내부 불균형 완화
- **LDS의 실용적 적용**: die→unit 집계 후 target 분포에 Gaussian smoothing 적용 → re-weighting에 활용
- **FDS는 딥러닝(TabNet, FT-Transformer)에 적합**: 트리 모델에는 직접 적용 어렵지만, Stage 7-A의 딥러닝 모델에 FDS 적용 가능
- **SMOTER/SMOGN 주의**: 고차원에서는 역효과 가능 → 우리도 8,640개 피처 환경에서 SMOGN 사용 시 주의

## 8-2. SMOGN: A Pre-processing Approach for Imbalanced Regression (2017)

### 논문 정보
- **저자**: Paula Branco, Luís Torgo, Rita P. Ribeiro (LIAAD-INESC TEC, University of Porto)
- **학회**: PMLR 74:36-50, LIDTA 2017 (Workshop on Learning with Imbalanced Domains)
- **URL**: http://proceedings.mlr.press/v74/branco17a/branco17a.pdf

### 연구 배경 및 목적
분류의 SMOTE를 회귀로 확장한 SmoteR의 잠재적 문제(먼 이웃 간 합성 → 노이즈)를 해결. **SMOGN = SmoteR + Gaussian Noise + Random Undersampling** 결합.

### 방법론
- **Relevance function φ(y)**: target 값의 중요도를 [0,1]로 매핑. 극단값(rare)이 높은 relevance
- **3단계 리샘플링**:
  1. **Random Undersampling**: normal cases(φ < threshold) 제거
  2. **SmoteR**: seed와 k-nearest neighbor가 "충분히 가까울 때" 선형 보간으로 합성 샘플 생성
  3. **Gaussian Noise**: seed와 neighbor가 "멀 때" Gaussian noise를 추가하여 합성 (SmoteR의 노이즈 위험 완화)
- 가까움/멀음 판단: seed와 neighbor 거리의 중앙값(또는 사용자 정의 threshold)

### 핵심 결과

**20개 데이터셋 × 4개 learner (MARS, SVM, RF, NNET)**:
- SMOGN이 **MARS와 RF에서 가장 큰 이점** (전체 560회 비교 중 wins > losses)
- SmoteR 단독 대비: SMOGN이 wins > losses (SMOGN만 유일)
- Gaussian Noise 단독 대비: SMOGN과 SmoteR 모두 wins > losses
- **SVM에서는 SMOGN 이점 적음**, NNET에서는 SmoteR과 유사

### 우리 프로젝트 적용 시사점
- **회귀용 오버샘플링의 기본 참고**: SMOTE는 분류 전용이므로, Y>0 서브셋의 불균형 완화에 SmoteR/SMOGN을 고려
- **트리 모델(RF, MARS)에서 효과적**: Stage 7-A의 RF, ExtraTrees에 SMOGN 전처리 적용 가능
- **SVM/NN에서는 주의**: 딥러닝 모델에는 LDS/FDS(8-1)가 더 적합
- **DIR 논문(8-1)의 경고**: 고차원에서 SMOGN이 역효과 가능 → 8,640개 피처 환경에서는 LDS re-weighting이 더 안전할 수 있음

## 8-3. Highly Imbalanced Regression with Tabular Data in SEP and Other Applications (2025)

### 논문 정보
- **저자**: Josias K. Moukpe, Philip K. Chan, Ming Zhang (Florida Institute of Technology)
- **학회**: arXiv:2509.16339 (2025)
- **URL**: https://arxiv.org/abs/2509.16339

### 연구 배경 및 목적
**Imbalance ratio > 1,000**인 극단적 불균형 회귀(tabular) 문제를 다룸. 태양 고에너지 입자(SEP) 강도 예측 등 5개 데이터셋에서 검증. MSE만으로는 불충분함을 지적하고 상관계수를 보조 loss로 추가하는 **CISIR** 제안.

### 방법론
- **CISIR = Correlation + Involution importance + Stratified sampling for Imbalanced Regression**
  1. **Weighted Pearson Correlation Coefficient (wPCC)**: MSE 외에 예측값-실제값 상관을 보조 loss로 추가
  2. **MDI (Monotonically Decreasing Involution) Importance**: 기존 inverse importance(볼록 함수만 가능)를 확장하여 볼록/선형/오목 함수 family 지원
  3. **Stratified Sampling**: mini-batch에 rare 인스턴스가 반드시 포함되도록 층화 샘플링
- Neural Network (MLP) 기반 구현

### 핵심 결과

**5개 데이터셋 평균 성능**:

| 방법 | MSE Rank (↓) | Pearson r Rank (↓) |
|------|-------------|-------------------|
| Vanilla | 4.4 | 5.0 |
| LDS+FDS | 3.4 | 3.6 |
| ConR | 3.8 | 3.4 |
| **CISIR** | **2.0** | **1.8** |

- CISIR이 5개 데이터셋 중 대부분에서 **MSE 최저 + 상관계수 최고**
- wPCC를 다른 방법(LDS, ConR)에 추가해도 성능 개선 → **범용 보조 loss로 활용 가능**
- MDI importance가 inverse/DenseLoss 등 기존 importance function 대비 우수

### 우리 프로젝트 적용 시사점
- **Y>0 서브셋의 극단적 skew 대응**: Y>0 값의 범위가 0.001~1.0으로 imbalance ratio가 매우 높음. CISIR의 stratified sampling + MDI importance 적용 가능
- **wPCC 보조 loss**: RMSE만 최적화하면 극단값(high health) 예측이 약할 수 있음. wPCC를 custom loss에 추가하면 "순위 보존" 효과
- **딥러닝 모델(TabNet, FT-Transformer)에 적용**: MLP 기반이므로 tabular DL에 직접 적용 가능

- **저자**: Nick Erickson, Jonas Mueller 외 (Amazon)
- **학회**: arXiv:2003.06505 (ICML 2020 AutoML Workshop)

### 연구 배경 및 목적
기존 AutoML이 CASH(Combined Algorithm Selection and Hyperparameter optimization)에 집중하는 반면, **multi-layer stacking + k-fold bagging**으로 시간을 더 효율적으로 사용하는 AutoML 프레임워크 제안. 50개 Kaggle/OpenML 데이터셋에서 다른 모든 AutoML을 압도.

- Out-of-fold 예측으로 stacking에 사용 → 데이터 누수 방지

**자동 전처리:**
- 컬럼 타입 자동 인식 (수치형, 범주형, 텍스트, datetime)

- 2개 Kaggle 대회에서 AutoGluon이 4시간 학습만으로 **99% 참가자 능가**
- Multi-layer stacking이 single model HPO보다 동일 시간 내 더 높은 성능

## 9-1. Revisiting Self-Training with Regularized Pseudo-Labeling for Tabular Data (2023)

### 논문 정보
- **저자**: Minwook Kim, Juseong Kim, Giltae Song (Pusan National University)
- **학회**: arXiv:2302.14013 (2023)
- **URL**: https://arxiv.org/abs/2302.14013

### 연구 배경 및 목적
Tabular 데이터에서 **semi-supervised learning**이 성공하지 못한 이유를 분석하고, **GBDT에도 적용 가능한** self-training + pseudo-labeling 방법을 제안. Computer Vision의 curriculum pseudo-labeling을 tabular에 도입하고, **likelihood 기반 정규화**로 pseudo-label 신뢰도를 개선.

### 방법론

**1. Curriculum Pseudo-Labeling (CPL):**
- 학습 초기에는 높은 confidence threshold → 점진적으로 낮춤 (쉬운 것부터 어려운 것 순)
- Fixed threshold pseudo-labeling(FPL)의 confirmation bias 문제 완화

**2. Regularized Pseudo-Labeling (R-CPL, R-FPL):**
- Confidence score에 **log-likelihood**를 곱하여 정규화
- `f(x) = c(x) × (λ × log_likelihood(x) + 1) / (λ + 1)`
- 높은 밀도 영역의 pseudo-label은 신뢰도 유지, 저밀도 영역은 신뢰도 감소 → cluster assumption 보장
- λ: likelihood 영향 강도 조절 (λ=0이면 기존 방법과 동일)

### 핵심 결과

**XGBoost + 12개 데이터셋**:

| 방법 | 평균 순위 (↓) |
|------|-------------|
| NONE (supervised only) | 4.8 |
| FPL (fixed threshold) | 3.3 |
| CPL (curriculum) | 3.3 |
| R-FPL (regularized fixed) | 2.0 |
| **R-CPL (regularized curriculum)** | **1.6** |

- R-CPL이 **12개 데이터셋 평균 순위 1위**, 대부분의 데이터셋에서 통계적으로 유의한 개선
- GBDT(XGBoost)뿐 아니라 TabNet, SAINT 등 딥러닝에도 효과적
- Pretraining + R-CPL 조합이 최고 성능 (TabNet에서 검증)

### 우리 프로젝트 적용 시사점
- **val/test X 데이터 활용**: validation(34,996 dies), test(34,996 dies)의 X 데이터에 pseudo-label 부여 → 학습 데이터 3배 확장
- **GBDT 호환**: XGBoost, LightGBM, CatBoost에 직접 적용 가능 (gradient descent 불필요)
- **R-CPL 적용 절차**:
  1. Train X+Y로 초기 모델 학습
  2. Val/Test X에 예측 → confidence 높은 샘플에 pseudo-label 부여
  3. Train + pseudo-labeled 데이터로 재학습
  4. 반복
- **TabNet self-supervised pretraining과 병행**: TabNet은 val/test X로 pretraining → R-CPL로 추가 개선

## 9-2. Stacking Ensemble Learning: Combining XGBoost, LightGBM, CatBoost, and AdaBoost with Random Forest Meta Model (2025)

### 논문 정보
- **저자**: Sindhu Wijaya Mulyo Nugroho (Educourse.id, Indonesia)
- **학회**: Research Square preprint (2025-10-30)
- **DOI**: https://doi.org/10.21203/rs.3.rs-7944070/v1
- **ORCID**: 0009-0002-1558-4574

### 연구 배경 및 목적
단일 부스팅 알고리즘(XGBoost, LightGBM, CatBoost, AdaBoost)은 각각 강점과 단점이 뚜렷하여 어느 하나가 모든 태스크에서 지배적이지 않다. 4개 부스팅을 **base learner**로 두고 **Random Forest를 meta-learner**로 사용하는 이중(boosting+bagging) 스태킹 구조를 제안. 네트워크 침입 탐지(Network Intrusion Detection) 태스크에서 검증한다.

### 데이터
- 5개 네트워크 침입 탐지 벤치마크 데이터셋 (Kaggle 기반)
  - CICIDS Collection, IDS Intrusion CSV, CSE-CIC-IDS2018 (cleaned/원본), USBIDS1
- Label: 정상(0) vs 공격(1, 2, 3 — DoS/DDoS/PortScan/Brute Force)
- Feature: flow-based 네트워크 메타 (패킷 수, duration, 패킷 크기, idle/active time 등)

### 방법론

**전체 구조:**
```
[XGBoost] [CatBoost] [LightGBM] [AdaBoost]   (Base 층 — Boosting)
       ↓ 각자 p_i(x) (확률 벡터) 출력
z(x) = [p_1(x), p_2(x), p_3(x), p_4(x)]     (meta-feature)
       ↓
  [Random Forest]                            (Meta 층 — Bagging)
       ↓
    ŷ = g(z(x))                              (최종 예측)
```

**수식:**
- Base 층: 각 모델 M_i가 확률 p_i(x) 생성
- Meta 층: `ŷ = g(z(x)) = (1/T) Σ_{t=1}^T h_t(z(x))` (RF T개 트리의 평균 / 분류는 majority vote)
- Breiman의 편향-분산 분해 `PE(f) = σ² + Bias²(f) + Var(f)`에 근거, boosting(저편향) + bagging(저분산)의 상보성 주장

**학습 절차:**
1. Data Preparation (CSV 로드, label encoding, 80:20 split)
2. 4개 base model 독립 학습
3. base 출력을 concat → meta-feature 생성
4. Random Forest meta-learner 학습 (hold-out 기반, CV는 명시적 서술만)
5. Test set에서 base 예측 → RF가 최종 레이블 반환

**의의: 일반적 스태킹과의 차이**
- 전형적 스태킹은 meta-learner로 Ridge/Logistic/선형 모델을 쓰지만, 이 논문은 **비선형 의존성과 feature importance 해석**을 얻기 위해 **Random Forest**를 meta로 채택
- RF meta의 근거: (1) bootstrap+random feature로 correlated base error에 강건, (2) base 예측 간 nonlinear 상호작용 포착, (3) feature importance로 "어떤 base가 어디서 기여했는지" 해석 가능

### 핵심 결과 (Accuracy)

| 모델 | Accuracy (%) | Vulnerability (%, 공격으로 분류 비율) |
|------|-------------|------------------------------------|
| XGBoost | 100.00 | 22.49 |
| CatBoost | 99.99 | 22.50 |
| LightGBM | 70.41 | 26.11 |
| AdaBoost | 99.96 | 22.49 |
| **Stacking (RF meta)** | **100.00** | 22.50 |

- Stacking meta가 XGBoost와 동률(100%)이며 단일 LightGBM 대비 29.6%p 향상
- XGBoost/CatBoost/AdaBoost가 거의 포화 성능 → 데이터셋이 상대적으로 선형 분리 가능한 구조
- LightGBM이 유독 낮은 것은 default 하이퍼파라미터 민감도 문제로 해석 (저자 언급)
- Stacking은 변동성(variance) 감소가 주 효과, 이미 포화된 베이스에서는 accuracy 추가 향상은 제한적

### 우리 프로젝트 적용 시사점

**직접 적용 가능한 것:**
- **Stage 7-C Stacking 설계에 RF meta 옵션 추가**: 기존 전략은 meta-learner를 Ridge/LightGBM으로 잡았으나, 본 논문은 RF meta의 이론적/실험적 근거 제시 → **Ridge vs LightGBM vs RF meta 3종 비교 실험**할 가치가 있음
- **RF meta의 장점이 우리 데이터에 잘 맞는 조건**:
  - base learner들의 예측 에러가 correlated일 때 → 우리는 LightGBM/XGBoost/CatBoost가 모두 GBDT 계열이라 correlated error가 예상됨 → RF meta가 더 robust할 수 있음
  - base 예측 간 nonlinear 상호작용 중요 → zero-inflated 특성상 P(Y>0)와 E[Y|Y>0]의 결합이 비선형
- **Two-Stage + Stacking 조합**: Stage 1(분류) + Stage 2(회귀) 각각의 출력을 base로 쌓고 RF meta로 통합하는 hybrid 구조 실험 가능
- **Base learner 다양성 확보**: XGBoost+LightGBM+CatBoost+AdaBoost 조합이 복수 논문(본 논문, 4-1, 4-4, 2-6)에서 언급된 표준 조합

**주의/한계 (데이터/태스크 특성 차이):**
- **본 논문은 분류(accuracy)**, 우리는 **회귀(RMSE)** — RF meta의 nonlinear aggregation은 회귀에서도 작동하지만, base 예측 분산이 RF 내 평균화로 과도하게 smooth되어 극단값(health max=1.0) 예측 성능이 떨어질 수 있음 → 회귀용으로는 `RandomForestRegressor` 사용 + **min_samples_leaf 작게** 설정 필요
- **결과가 거의 포화(100%)** 하여 "stacking이 얼마나 기여했는가"가 불분명. 우리 문제는 포화와 거리가 멀기 때문에 RF meta의 효과 크기는 재검증 필요
- **CV 기반 스택 누수 방지가 방법론 절에서 언급만 되고 실제 구현은 80:20 단순 split** — 우리는 K-fold OOF 방식으로 **반드시** base 예측을 생성해야 누수 없음
- **AdaBoost의 기여는 명확치 않음** — 본 논문에서도 단일 성능이 XGBoost/CatBoost와 비슷하여 ensemble diversity 기여가 적음 → 우리 baseline에는 AdaBoost 우선순위 낮음
- **Research Square preprint**로 peer-review 전이며, LightGBM 70.41%는 튜닝 부재 때문일 가능성이 높음 → 단일 실험 결과를 강한 근거로 삼기보다는 **설계 가이드** 수준으로 활용

**실험 제안:**
1. 기존 Stage 7-C Stacking(Ridge meta) 기준선 확보
2. 같은 base 구성에서 **RF meta(n_estimators=500, max_depth=None, min_samples_leaf=5)**로 교체 실험
3. **LightGBM meta**와 3자 비교 → val RMSE 기준 최적 meta 선정
4. CV fold OOF 예측으로 meta 학습 데이터 생성(누수 방지)

## 10-1. Attention-based Deep Multiple Instance Learning (2018)

### 논문 정보
- **저자**: Maximilian Ilse, Jakub M. Tomczak, Max Welling (University of Amsterdam)
- **학회**: ICML 2018 (Proceedings of the 35th International Conference on Machine Learning)
- **URL**: https://arxiv.org/abs/1802.04712

### 연구 배경 및 목적
Multiple Instance Learning(MIL)에서 bag 내 인스턴스들을 어떻게 집계할 것인가의 문제. 기존 max/mean pooling 대신 **attention 기반 가중 평균**을 제안하여 (1) bag 분류 성능 향상, (2) 핵심 인스턴스(key instance) 식별의 해석력 확보.

### 방법론

**MIL 문제 구조:**
- Bag X = {x₁, x₂, ..., x_K}: K개의 인스턴스로 구성
- Bag label Y: 하나의 레이블 (인스턴스 레이블은 미관측)
- Standard MIL assumption: Y=1 ↔ bag 내 하나 이상의 positive instance 존재

**Attention-based Aggregation:**
```
z = Σ_k a_k · h(x_k)    (가중 평균)
a_k = exp(w^T tanh(V h(x_k))) / Σ_j exp(w^T tanh(V h(x_j)))   (attention weight)
```
- h(x_k): 인스턴스 임베딩 (CNN 또는 FC로 학습)
- a_k: 인스턴스 k의 attention weight (모든 k에 대해 합=1)
- 학습 가능한 파라미터 w, V로 "어떤 인스턴스가 bag label에 중요한지" 자동 학습

**Gated Attention:**
- `tanh`에 `sigmoid` gate를 곱하여 element-wise control 추가
- `a_k = exp(w^T (tanh(V h_k) ⊙ sigm(U h_k))) / Σ_j ...`

### 핵심 결과

**MNIST-bags (합성 데이터)**:

| 방법 | AUC (bag size=10, 50 bags) |
|------|---------------------------|
| MI-SVM | ~0.82 |
| Instance-based max | ~0.88 |
| Embedding-based max | ~0.90 |
| **Attention-based** | **~0.96** |

- Small sample 환경에서 attention이 다른 방법 대비 **6~14%p AUC 향상**
- Attention weight가 key instance(숫자 '9')를 정확히 지목 (해석력 확인)

**조직병리학 이미지 (Breast Cancer, Colon Cancer)**:
- Attention이 기존 MIL 대비 동등~우수한 AUC
- Attention weight로 ROI(Region of Interest) 자동 하이라이트 가능

### 우리 프로젝트 적용 시사점
- **Die→Unit 집계에 Attention MIL 적용**: 현재 mean/std/max 등 수동 통계로 4개 die를 unit으로 집계하지만, Attention MIL로 "어떤 die가 unit 불량에 가장 기여하는지" 학습 가능
- **구현 구조**:
  - Bag = 1 unit (4개 die)
  - Instance = 1 die의 WT feature 벡터 (960개)
  - Bag label = health (회귀로 확장 필요)
  - Attention weight → 어떤 die가 불량 원인인지 해석
- **한계**: 원논문은 분류만 다루므로 회귀 확장 필요 (loss를 MSE로 변경, output을 continuous로)
- **기대 효과**: 수동 집계(mean, std 등)로 놓치는 die 간 복잡한 상호작용을 학습할 수 있음. 특히 "하나의 die만 불량이면 unit이 불량"인 패턴을 max pooling보다 정교하게 포착

## 10-2. Industrial Multi-Machine Data Aggregation, AI-Ready Data Preparation, and ML for Virtual Metrology (2025)

### 논문 정보
- **저자**: Feiyang Ou 외 (UCLA, Seagate Technology)
- **학회**: Digital Chemical Engineering (Elsevier), Vol. 15, 100242 (2025)
- **URL**: https://www.sciencedirect.com/science/article/pii/S2772508125000262

### 연구 배경 및 목적
반도체 웨이퍼 및 슬라이더 생산에서 **다중 장비(multi-machine)의 운영 데이터를 집계**하고, **AI-Ready 데이터로 전처리**한 뒤 ML로 virtual metrology(품질 예측)를 수행하는 end-to-end 파이프라인.

### 방법론

**Data-first 전략 (AI-Ready Data):**
1. **Multi-machine data aggregation**: 같은 공정의 서로 다른 장비 데이터를 통합
   - Feature alignment: 장비별로 동일 물리량이지만 다른 센서/명명 규칙 → 통일
   - **Separated scaling**: 장비별로 별도 스케일링 후 합산 (일괄 스케일링보다 우수)
2. **PCA**: 차원 축소로 과적합 방지 + 노이즈 제거 (원 100+ 피처 → 10~20 PC)
3. **Data augmentation**: Linear Mixup으로 소량 데이터 증강 (슬라이더 생산)
4. **Transfer Learning**: SGD로 실시간 모델 업데이트 (공정 drift 대응)

**모델 비교:**
- XGBoost vs. FNN (Feedforward Neural Network)
- 분류(PASS/FAIL) + 회귀(milling depth)

### 핵심 결과

**Wafer 생산 (분류)**:

| 모델 | ROC-AUC |
|------|---------|
| FNN (단일 장비) | 0.87 |
| FNN (multi-machine, separated scaling) | 0.93 |
| **XGBoost (multi-machine, separated scaling)** | **0.95** |

**Slider 생산 (회귀)**:

| 모델 | R² | MAE |
|------|-----|-----|
| FNN baseline | 0.52 | 1.23 |
| FNN + Mixup + Transfer Learning | 0.78 | 0.71 |
| **XGBoost + PCA** | **0.82** | **0.65** |

- **Multi-machine 데이터 집계 + separated scaling**이 단일 장비 대비 AUC 0.08 향상
- XGBoost이 FNN 대비 **학습 속도, 자원 효율, 성능** 모두 우수
- PCA가 과적합 방지에 효과적 (원 피처 직접 사용 대비)
- Linear Mixup 데이터 증강이 소량 데이터에서 효과적

### 우리 프로젝트 적용 시사점
- **Separated scaling 전략**: 로트(run_id)별로 WT feature의 스케일이 다를 수 있음 → 로트별 별도 스케일링 후 합산 (Stage 4.5와 연관)
- **Multi-machine ≈ Multi-lot**: 우리 데이터에서 각 lot/wafer의 장비 조건이 다를 수 있으므로, lot별 데이터를 별도 스케일링+집계하는 전략 적용 가능
- **XGBoost의 일관된 우수성**: 산업용 tabular virtual metrology에서도 XGBoost > NN 확인
- **Linear Mixup**: train 데이터 증강으로 시도 가능 (단, Y=0 비율 유지에 주의)
