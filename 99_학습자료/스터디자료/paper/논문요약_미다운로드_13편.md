# 1. 반도체 WT / 수율 예측 (5편)

## 1-1. A Machine Learning-based Approach for Failure Prediction at Cell Level based on WAT Parameters (2021)

### 논문 정보
- **제목**: A Machine Learning-based Approach for Failure Prediction at Cell Level based on Wafer Acceptance Test Parameters
- **저자**: Xiang Chen, Yi Zhao, Hongliang Lü, Xiaoqiang Shao, Cheng Chen, Yu Huang (소속: 원문 확인 필요 — Huawei Technologies로 추정되나 웹 공개 정보에서 소속 명시 미확인)
- **연도**: 2021
- **학회**: IEEE Microelectronics Design & Test Symposium (MDTS)
- **URL**: https://ieeexplore.ieee.org/document/9476151

### 연구 배경 및 목적
WAT(Wafer Acceptance Test) 데이터와 후속 진단 보고서(diagnosis report) 간의 관계가 충분히 탐색되지 않았다. 산업용 WAT 데이터셋을 사용하여 **셀 레벨에서의 불량 예측 프레임워크**를 제안한다.

### 방법론
- **피처 선택**: RFECV(Recursive Feature Elimination with Cross Validation), Pearson 상관 분석
- **클래스 불균형 처리**: SMOTE (Synthetic Minority Oversampling Technique)
- **모델**: XGBoost 등 ML 모델 비교
- 학습 프레임워크에 피처 선택 + 불균형 처리를 통합

### 핵심 결과
- Accuracy **0.95+**, F1-score **0.95+**, AUROC **0.95+** 달성
- 산업용 실데이터에서 검증

### 우리 프로젝트 적용 시사점
- **WAT → 불량 예측 구조가 우리와 가장 유사**: WT 데이터에서 셀/die 레벨 불량을 예측하는 프레임워크
- **RFECV + SMOTE + XGBoost** 파이프라인이 산업 데이터에서 효과적임을 확인
- 산업 실데이터에서 0.95+ 성능은, 우리 SK Hynix 데이터에서도 WT가 Field Health에 대한 예측력을 가진다는 간접 근거

## 1-2. Using Wafer Map Features to Better Predict Die-Level Failures in Final Test (2015)

### 논문 정보
- **제목**: Using Wafer Map Features to Better Predict Die-Level Failures in Final Test
- **저자**: Seokho Kang, Sungzoon Cho (서울대), **Daewoong An, Jaeyoung Rim (SK Hynix DRAM Development Division)**
- **연도**: 2015
- **학회**: IEEE Transactions on Semiconductor Manufacturing, Vol. 28, No. 3, pp. 431-437
- **URL**: https://ieeexplore.ieee.org/document/7120993

### 연구 배경 및 목적
불량 반도체 die가 wafer 레벨 테스트를 통과하고 최종 조립 테스트에서 실패하는 문제를 해결. **웨이퍼 맵에서 도출한 공간 피처**로 이러한 잠재 불량을 사전 예측.

### 방법론 (핵심)
**4가지 웨이퍼 맵 파생 변수:**
1. **웨이퍼 중심으로부터의 거리 (distance from wafer center)** — edge effect 포착
2. **해당 die 위치의 과거 최종 수율 (historical final yield at die position)** — 위치별 불량 이력
3. **인접 die의 wafer test 불량률 (failure rate among adjacent dies)** — 클러스터링 효과
4. **웨이퍼 맵 내 이상 패턴 (abnormal wafer map patterns)** — 전체 맵 수준 이상 감지

**모델**: Random Forest 분류기

### 핵심 결과
- 4가지 파생 변수 사용 시 기존 WT 피처만 사용한 경우 대비 **통계적으로 유의한 예측 성능 향상**
- SK Hynix DRAM 실생산 데이터에서 검증

### 우리 프로젝트 적용 시사점

> **이 논문은 우리 프로젝트에 가장 직접적으로 관련된 논문이다. SK Hynix 공저.**

| 파생 변수 | 우리 프로젝트 대응 |
|-----------|------------------|
| 웨이퍼 중심 거리 | `radial_dist = sqrt(die_x² + die_y²)` (Stage 4.5) |
| die 위치별 과거 수율 | `run_id`별 해당 위치의 WT 통계 (로트 집계 피처) |
| 인접 die 불량률 | 같은 unit 내 다른 die의 WT 값 (position별 집계) |
| 웨이퍼 맵 이상 패턴 | 로트/웨이퍼 단위 이상 탐지 피처 |

- **CLAUDE.md Stage 4.5 전략의 학술적 근거**: die_x, die_y, radial_dist, is_edge 피처가 정확히 이 논문의 변수 1에 대응
- **인접 die 정보**: unit 내 4개 die의 WT 피처를 std, range로 집계하면 변수 3의 효과를 간접 포착
- SK Hynix 데이터에서 검증된 결과이므로 **우리 데이터에도 직접 적용 가능성 높음**

## 1-3. A Wafer Map Yield Prediction Based on Machine Learning for Productivity Enhancement (2019)

### 논문 정보
- **제목**: A Wafer Map Yield Prediction Based on Machine Learning for Productivity Enhancement
- **저자**: S.-J. Jang, J.-S. Kim, T.-W. Kim, H.-J. Lee, S. Ko (소속: 원문 확인 필요 — Samsung Electronics 추정이나 웹 공개 정보에서 소속 명시 미확인)
- **연도**: 2019
- **학회**: IEEE Transactions on Semiconductor Manufacturing, Vol. 32, No. 4, pp. 400-407
- **URL**: https://ieeexplore.ieee.org/document/8856232

### 연구 배경 및 목적
반도체 웨이퍼 수율을 **fabrication 이전에 예측**하여 생산성을 향상시키는 모델 개발. 새로운 wafer map의 수율을 사전 예측하여 wafer map 생산성을 평가·최적화하는 것이 목표. gross dies, shot counts, lithography throughput, MFO(mask field occupancy), 가격 등 다양한 요인을 고려.

### 방법론
- **공간 관계 활용**: die 위치 간의 공간적 관계, die 크기, die-level 수율 변동을 ML로 모델링
- **wafer map 수율 예측**: 공정 정보 없이 wafer map 설계 자체에서 수율을 예측하는 모델
- 관련 논문(같은 저자, IEEE 2018, doc 8373137)에서는 deep learning 기반 모델도 제안

### 핵심 결과
- 제안 모델이 기존 대비 생산성 향상 달성 (구체 수치는 원문 확인 필요)
- 반도체 실생산 데이터에서 검증

### 우리 프로젝트 적용 시사점
- **wafer map 설계 단계에서 수율을 예측**하는 접근으로, 우리 프로젝트(WT→Field Health 예측)와 예측 대상은 다르지만 공간 관계 활용 방법론이 참고 가치 있음
- **공간 관계가 수율 예측에 핵심**: die 위치의 공간 패턴이 수율과 관계. die_x, die_y, radial_dist 피처의 중요성 뒷받침
- 반도체 제조의 물리적 원리는 공통 → SK Hynix 데이터에도 적용 가능

## 1-4. Wafer Yield Prediction Using Derived Spatial Variables (2017)

### 논문 정보
- **제목**: Wafer Yield Prediction Using Derived Spatial Variables
- **저자**: Hang Dong, Nan Chen (National University of Singapore), Kaibo Wang (Tsinghua University)
- **연도**: 2017
- **학회**: Quality and Reliability Engineering International
- **URL**: https://onlinelibrary.wiley.com/doi/abs/10.1002/qre.2192

### 연구 배경 및 목적
웨이퍼 맵에서 결함 클러스터의 공간 분포를 **Fused LASSO**로 모델링하여 수율 예측. 신뢰할 수 없는 칩(unreliable chips)이 공간 클러스터를 형성하는 경향을 명시적으로 활용.

### 방법론
- **Fused LASSO**: L1 정규화 + 인접 계수 간 차이에 대한 L1 정규화 → 공간적으로 인접한 위치의 예측값이 유사해지도록 강제 (공간 평활화)
- 결함 클러스터 분포 특성(위치, 크기, 밀도)을 파생 변수로 생성
- 기존 단일 수율 모델 대비 공간 클러스터 정보 추가

### 핵심 결과
- 클러스터 특성을 명시적으로 모델링하면 기존 방법 대비 **유의한 수율 예측 향상**
- 결함 클러스터의 공간 분포가 수율에 직접적 영향을 미침을 실증

### 우리 프로젝트 적용 시사점
- **공간 클러스터링 피처**: unit 내 4개 die의 공간 분포 특성 (die 좌표의 mean, std, range)이 클러스터 정보를 간접 포착
- **Fused LASSO 아이디어**: 피처 간 공간 연속성을 활용하는 정규화 → 우리 데이터에서 인접 피처(예: X100, X101)가 유사한 물리량을 측정할 수 있으므로, 그룹 정규화 고려 가능

# 2. Zero-Inflated 모델 이론 (1편)

## 2-1. Zero-Inflated Poisson Regression, with an Application to Defects in Manufacturing (1992)

### 논문 정보
- **제목**: Zero-Inflated Poisson Regression, with an Application to Defects in Manufacturing
- **저자**: Diane Lambert (AT&T Bell Laboratories)
- **연도**: 1992
- **학회**: Technometrics, Vol. 34, No. 1, pp. 1-14 (인용 3,856+회)
- **URL**: https://www.tandfonline.com/doi/abs/10.1080/00401706.1992.10485228

### 연구 배경 및 목적
제조 공정에서 결함 수(defect count) 데이터는 **excess zeros**(과도한 0값)가 발생한다. 표준 Poisson 회귀로는 이를 적절히 모델링할 수 없다. **Zero-Inflated Poisson(ZIP) 회귀** 모델을 제안하여, 0이 발생하는 두 가지 메커니즘을 분리 모델링한다.

### 방법론 (ZIP 모델의 수학적 정의)

**핵심 아이디어**: 데이터의 0은 두 가지 소스에서 발생한다:
1. **구조적 zero (structural zero)**: 시스템이 "완벽한 상태(perfect state)"에 있어 결함이 발생할 수 없는 경우
2. **우연적 zero (sampling zero)**: 시스템이 "불완벽한 상태"이나 우연히 결함이 0인 경우

**확률 모델:**
```
P(Y=0) = π + (1-π) · exp(-λ)          [구조적 zero + 우연적 zero]
P(Y=k) = (1-π) · exp(-λ) · λᵏ / k!    (k = 1, 2, 3, ...)
```

여기서:
- `π` = P(완벽한 상태) — 결함이 원천적으로 불가능한 확률
- `λ` = 불완벽한 상태에서의 기대 결함 수

**회귀 구조 (두 개의 동시 모델):**
```
logit(π) = z'γ     [완벽 상태 여부를 설명하는 공변량 z와 계수 γ]
log(λ)   = x'β     [결함 수를 설명하는 공변량 x와 계수 β]
```

**추정 방법**: EM(Expectation-Maximization) 알고리즘
- E-step: 각 관측치가 "완벽 상태"에서 왔을 사후 확률 계산
- M-step: 가중 GLM으로 γ와 β를 갱신

**제조업 적용 사례**: 인쇄 회로 기판(PCB) 솔더링 결함
- 장비 정렬(alignment)이 잘 되면 → 완벽 상태 (π, 결함 0 보장)
- 장비 정렬이 불완전하면 → 불완벽 상태 (Poisson 분포로 결함 발생)

### 핵심 결과
- 표준 Poisson보다 **overdispersion을 크게 감소**시킴
- 0의 과다를 명시적으로 모델링하여 비-zero 부분의 예측 정확도 향상
- 3,856+회 인용으로 zero-inflated 모델링의 표준 참고문헌으로 정착

### 우리 프로젝트 적용 시사점

| 개념 | 우리 프로젝트 대응 |
|------|------------------|
| **구조적 zero** (완벽한 상태) | Y=0인 unit 중 "진짜 양품" — Field에서 불량이 발생할 수 없는 unit |
| **우연적 zero** (우연히 0) | Y=0인 unit 중 "잠재 불량" — 아직 드러나지 않았지만 불량 위험이 있는 unit |
| **두 모델의 분리** | **Two-Stage의 이론적 근거**: Stage 1(분류)이 π를 학습, Stage 2(회귀)가 λ를 학습 |
| **EM 알고리즘** | Stage 1의 확률 출력 P(Y>0)을 Stage 2의 가중치로 사용하는 soft 접근법도 가능 |

- **우리 데이터의 Y=0 (70.8%)이 모두 동질적이지 않다**: 일부는 진짜 양품, 일부는 잠재 불량. Two-Stage 모델이 이 구분을 학습
- Lambert(1992)의 count 데이터 한정 모델을 **연속 데이터에 확장한 것이 논문요약.md의 2-1(Hybrid ML)**

# 3. 핵심 ML 모델 원논문 (4편)

## 3-1. XGBoost: A Scalable Tree Boosting System (2016)

### 논문 정보
- **제목**: XGBoost: A Scalable Tree Boosting System
- **저자**: Tianqi Chen, Carlos Guestrin (University of Washington)
- **연도**: 2016
- **학회**: KDD'16 (ACM SIGKDD), pp. 785-794
- **URL**: https://dl.acm.org/doi/10.1145/2939672.2939785 / arXiv:1603.02754

### 핵심 기술

**1. 정규화된 목적 함수 (Regularized Objective)**
```
Obj = Σ l(yi, ŷi) + Σ Ω(fk)
Ω(f) = γT + ½λ||w||²
```
- `l`: 손실 함수 (MSE, logloss 등)
- `Ω`: 정규화 항 (트리 복잡도 패널티)
- `γ`: 리프 노드 수(T)에 대한 패널티 → 과적합 방지
- `λ`: 리프 가중치(w)의 L2 정규화 → 과적합 방지

**2. Sparsity-Aware Algorithm**
- 결측값과 희소 데이터를 효율적으로 처리
- 결측값이 있는 샘플을 자동으로 최적 방향으로 분기 (default direction learning)
- 별도 imputation 없이 결측값 처리 가능

**3. Weighted Quantile Sketch**
- 근사 분할(approximate split finding)을 위한 가중 분위수 알고리즘
- 대규모 데이터에서 정확한 분할점을 효율적으로 찾음
- 분산 환경에서도 merge 가능한 알고리즘

**4. 시스템 최적화**
- **Column Block**: 데이터를 열(column) 기준으로 정렬하여 캐시 효율 극대화
- **Cache-Aware Access**: 메모리 접근 패턴 최적화
- **Out-of-Core Computation**: 메인 메모리 초과 데이터도 디스크 I/O로 처리
- **Data Compression + Sharding**: 분산 학습 지원

**5. 추가 기법**
- **Shrinkage (학습률)**: 각 트리의 기여도를 η로 축소 → 과적합 방지
- **Column Subsampling**: Random Forest의 피처 서브샘플링을 부스팅에 도입 → 과적합 방지 + 속도 향상

### 우리 프로젝트 적용 시사점
- **결측값 자동 처리**: 우리 데이터의 1,087개 피처 모두에 결측 존재. XGBoost는 별도 imputation 없이도 학습 가능 (성능 비교 실험 가치)
- **정규화 파라미터 γ, λ**: ~6,700개 고차원 피처에서 과적합 방지에 핵심. Optuna HPO로 최적화 필수
- **Column Subsampling**: 고차원 데이터에서 colsample_bytree를 0.3~0.7로 설정하면 속도+성능 동시 향상
- Kaggle/산업 경진대회에서 가장 많이 사용되는 모델 → 우리 경진대회에서도 핵심 baseline

## 3-2. LightGBM: A Highly Efficient Gradient Boosting Decision Tree (2017)

### 논문 정보
- **제목**: LightGBM: A Highly Efficient Gradient Boosting Decision Tree
- **저자**: Guolin Ke, Qi Meng, Thomas Finley, Taifeng Wang, Wei Chen, Weidong Ma, Qiwei Ye, Tie-Yan Liu (Microsoft Research)
- **연도**: 2017
- **학회**: NeurIPS 2017 (NIPS), pp. 3146-3154
- **URL**: https://proceedings.neurips.cc/paper/2017/hash/6449f44a102fde848669bdd9eb6b76fa-Abstract.html

### 핵심 기술

**1. GOSS (Gradient-based One-Side Sampling)**
- 그래디언트가 큰 샘플(잘 맞추지 못하는 샘플)은 모두 유지
- 그래디언트가 작은 샘플(이미 잘 맞추는 샘플)은 랜덤 서브샘플링
- 정보 손실을 최소화하면서 학습 데이터 크기를 대폭 축소
- **효과**: 학습 속도 향상, 정확도 유지

**2. EFB (Exclusive Feature Bundling)**
- 상호 배타적인 피처들(동시에 non-zero가 아닌 피처)을 하나로 번들링
- 고차원 희소 데이터에서 실질적 피처 수를 대폭 축소
- 그래프 coloring 알고리즘으로 번들 결정
- **효과**: 피처 수 감소 → 분할점 탐색 비용 절감

**3. Leaf-wise (Best-first) Tree Growth**
- 기존 GBDT의 level-wise(층별) 성장 대신, **가장 큰 loss 감소를 주는 리프를 우선 분할**
- 같은 리프 수에서 더 낮은 loss 달성
- 과적합 위험이 있으므로 max_depth로 제어

**4. Histogram-based Algorithm**
- 연속 피처를 이산 bin으로 히스토그램화
- 분할점 탐색 시 bin 단위로 계산 → O(#data × #features) → O(#bins × #features)
- 메모리 효율적

### 핵심 결과
- 기존 GBDT 대비 **20배 이상 학습 속도 향상**, 거의 동일한 정확도
- 대규모 데이터셋에서 XGBoost보다 빠르고 메모리 효율적

### 우리 프로젝트 적용 시사점
- **~44K × ~6,700 데이터에 최적**: GOSS + EFB가 대규모 피처에서 속도와 정확도를 동시에 확보
- **EFB가 우리 데이터에 특히 유용**: 1,087개 WT 피처 중 상당수가 희소하거나 상호 배타적일 수 있음 → LightGBM이 자동으로 번들링
- **Leaf-wise 성장**: 비정보적 피처가 많은 우리 데이터에서, loss 감소가 큰 분할을 우선 수행하므로 효율적
- **Optuna HPO 핵심 파라미터**: `num_leaves`, `max_depth`, `learning_rate`, `min_child_samples`, `colsample_bytree`, `reg_alpha`, `reg_lambda`
- **Category 피처 네이티브 지원**: 이산형 피처(고유값 ≤20인 391개)를 원-핫 인코딩 없이 직접 처리 가능

## 3-3. TabNet: Attentive Interpretable Tabular Learning (2021)

### 논문 정보
- **제목**: TabNet: Attentive Interpretable Tabular Learning
- **저자**: Sercan O. Arik, Tomas Pfister (Google Cloud AI)
- **연도**: 2021 (arXiv 2019, AAAI 2021)
- **학회**: AAAI Conference on Artificial Intelligence, Vol. 35(8), pp. 6679-6687
- **URL**: https://ojs.aaai.org/index.php/AAAI/article/view/16826 / arXiv:1908.07442

### 핵심 기술

**1. Sequential Attention (순차적 주의 메커니즘)**
- 각 decision step에서 **어떤 피처에 집중할지 동적으로 선택**
- Sparsemax 활성화 함수로 소수의 피처만 선택 (sparse feature selection)
- 각 step의 피처 선택 결과가 다음 step에 반영 (sequential dependency)

**2. Feature Reusage (피처 재사용)**
- 이전 step에서 선택된 피처 정보를 다음 step에 전달
- Relaxation factor로 동일 피처의 반복 선택 정도를 조절

**3. Attentive Transformer**
```
Step 1: 피처 서브셋 A1 선택 → 처리 → 부분 출력
Step 2: 피처 서브셋 A2 선택 → 처리 → 부분 출력
...
Step N: 피처 서브셋 AN 선택 → 처리 → 부분 출력
최종 출력 = Σ 부분 출력
```

**4. Self-Supervised Learning**
- Encoder-Decoder 구조로 마스킹된 피처를 복원하는 사전학습
- 라벨 없는 데이터를 활용하여 피처 표현 학습
- **최초로 테이블 데이터에서 자기지도학습의 효과를 보임**

**5. Interpretability (해석 가능성)**
- 각 decision step의 attention mask → 어떤 피처가 예측에 기여했는지 시각화
- Global feature importance = 모든 샘플의 attention mask 평균

### 핵심 결과
- Forest Cover Type, Poker Hand, Sarcos, Higgs 등에서 기존 DNN 및 일부 트리 모델 대비 우수
- 자기지도학습으로 라벨 부족 시 성능 향상
- 단, 후속 연구(Shwartz-Ziv 2021, Grinsztajn 2022)에서 XGBoost 대비 일관된 우위를 보이지 못함

### 우리 프로젝트 적용 시사점
- **피처 선택 내장**: ~6,700개 피처에서 자동으로 중요 피처를 선택. 별도 피처 선택 없이도 작동
- **해석 가능성**: attention mask로 어떤 WT 피처가 RCC 예측에 기여하는지 파악 → 공정 개선 인사이트
- **자기지도학습**: validation/test split의 X 데이터(Y 없음)를 사전학습에 활용 가능. 우리 데이터는 train 60% + val+test 40%(Y 비공개)이므로 이점이 있을 수 있음
- **단독 사용보다 앙상블 구성원으로 활용**: 논문요약.md 4-1의 결론에 따라, XGBoost와 앙상블할 때 가장 효과적

## 3-4. Revisiting Deep Learning Models for Tabular Data / FT-Transformer (2021)

### 논문 정보
- **제목**: Revisiting Deep Learning Models for Tabular Data
- **저자**: Yury Gorishniy, Ivan Rubachev, Valentin Khrulkov, Artem Babenko (Yandex Research)
- **연도**: 2021
- **학회**: NeurIPS 2021
- **URL**: https://proceedings.neurips.cc/paper/2021/hash/9d86d83f925f2149e9edb0ac3b49229c-Abstract.html / arXiv:2106.11959
- **코드**: https://github.com/yandex-research/rtdl-revisiting-models

### 핵심 기술

**1. Feature Tokenizer (피처 토큰화)**
- 각 수치형/범주형 피처를 개별 토큰(임베딩 벡터)으로 변환
- 수치형: `ti = bi + xi · wi` (선형 임베딩)
- 범주형: lookup table 임베딩
- **핵심**: 피처별 독립 임베딩 → 피처의 원래 의미(방향성)를 보존 → 회전 불변성을 깨뜨림

**2. Transformer Backbone**
- Feature Tokenizer의 출력 토큰들에 **Multi-Head Self-Attention** 적용
- 피처 간 복잡한 상호작용(interaction)을 attention으로 포착
- [CLS] 토큰을 추가하여 최종 예측에 사용

**3. ResNet Baseline**
- 논문에서 ResNet-like 아키텍처(MLP + skip connections)가 **강력한 기본 baseline**임을 발견
- 기존 연구에서 과소평가되었던 단순 아키텍처의 재평가

### 핵심 결과
- FT-Transformer가 **대부분의 테이블 데이터 태스크에서 다른 딥러닝 모델을 능가**
- 그러나 **GBDT와 비교하면 "보편적으로 우월한 해법은 없다"** — 데이터에 따라 다름
- Grinsztajn et al.(2022)에서 **딥러닝 중 유일하게 트리와 경쟁 가능**한 모델로 인정

### 우리 프로젝트 적용 시사점
- **Feature Tokenizer가 핵심**: 각 피처를 독립 토큰으로 변환하여 피처별 의미를 보존. 이것이 MLP보다 테이블 데이터에 적합한 이유 (Grinsztajn 2022의 원인 분석 3번)
- **Attention의 quadratic scaling 문제**: 피처 수 K에 대해 O(K²) 복잡도 → ~6,700개 피처에서는 비현실적. **Boruta 등으로 ~300~500개로 줄인 후 적용** 필수
- **앙상블에서 다양성 확보**: LightGBM/XGBoost와 FT-Transformer는 다른 inductive bias를 가지므로, 앙상블 시 다양성 확보에 효과적
- **코드 공개**: GitHub에서 즉시 사용 가능

| 모델 | 최고 성능 횟수 (111개 중) |
|------|------------------------|
| AutoGluon (앙상블) | **39** (35%) |
| SVM | 10 (9%) |

**트리 기반 앙상블(TE) 모델 한정 순위 (Table 3 — TE 7종 내 비교):**

| 모델 | TE 내 최고 성능 횟수 |
|------|---------------------|

| XGBoost | 10/111 |

> **주의**: Table 3(TE 한정)과 Table 4(전체 20개 모델)의 수치가 다름. 전체 비교 시 CatBoost는 7/111, 트리 모델끼리만 비교하면 19/111.

→ **"랜덤 테이블 데이터에서는 ML(트리) 모델이 안전한 선택"** — 단, 전체 2위는 SVM(10/111)으로, SVM이 특정 데이터셋에서 강세를 보이나 평균 성능은 낮음

**딥러닝이 트리를 이기는 조건 (메타학습, 86.1% 정확도):**

| 조건 | DL 유리 |
|------|---------|
| 행 수 적음 + 열 수 많음 | O |
| 높은 첨도(kurtosis) | **강하게** O |
| 분류 문제 | O (회귀보다) |
| Long-tailed 분포 | O |

**메타학습 모델**: 20개 메타피처(행수, 열수, 첨도, 엔트로피, 왜도, PCA 성분 수, 평균 Pearson 상관 등)로 ML vs DL 우위를 예측. H2O-DL 모델이 86.1% 정확도, AUC 0.78 달성.

### 우리 프로젝트 적용 시사점

**우리 데이터 메타피처 분석:**

| 메타피처 | 우리 데이터 | DL 유리? |
|---------|-----------|---------|
| 행 수 | ~26K (train unit) | 중간 → 약간 ML 유리 |
| 열 수 | ~6,700 (집계 후) | 많음 → **DL 유리** |
| 첨도 | 높음 (왜도 |skew|>2: 314개) | **DL 유리** |
| 과제 유형 | 회귀 | ML 유리 |
| 타겟 분포 | zero-inflated, long-tailed | **DL 유리** |
| 피처-타겟 상관 | 극히 낮음 (max 0.037) | 불확실 |

→ **혼합 신호**: 열 수와 첨도는 DL 유리이지만, 회귀 과제와 중간 샘플 수는 ML 유리. **양쪽 모두 시도하되 트리를 기본으로, DL을 앙상블 보조로 사용**하는 전략이 적합.

- **TabNet은 111개 중 0회 최고**: 단독 사용 부적합 확인. 앙상블 구성원으로만 가치
- **CatBoost 고려**: 트리 모델 한정 비교(Table 3)에서 19/111로 XGBoost(10/111)보다 자주 최고. 전체 비교(Table 4)에서도 CatBoost(7) > XGBoost(3). 우리 baseline에 CatBoost 포함 타당
- **AutoGluon 활용**: 자동 앙상블 프레임워크가 35%에서 최고 → 빠른 baseline 구축에 활용 가능
